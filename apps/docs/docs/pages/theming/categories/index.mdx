# Categories [Challenge category styling and configuration]

Each challenge category in rCTF has a dedicated color palette, icon, and display name configured in `apps/web/src/lib/utils/categories.ts`. The system uses dynamic CSS variable injection to apply category-specific colors to components. A category's color key (e.g., `red`) maps to CSS variables like `--category-foreground-l0{:css}`, which components reference via Tailwind classes such as `text-category-foreground-l0`.

Category aliases (`binary` → `pwn`, `rev` → `reverse`, `cryptography` → `crypto`) allow flexible naming while maintaining consistent styling.

:::note[How category styling works]
The `getCategoryStyle(color){:ts}` function generates inline CSS that binds a color's variables to generic category tokens:

```ts
getCategoryStyle('red')
// Returns:
// "--category-foreground-l0: var(--foreground-red-l0); 
//  --category-foreground-l1: var(--foreground-red-l1); 
//  --category-background-l0: var(--background-red-l0); 
//  --category-background-l1: var(--background-red-l1); 
//  --category-background-l1-hover: var(--background-red-l1-hover);"
```

Components apply this style to a container, then use generic classes:

```svelte
<div style={getCategoryStyle(config.color)}>
  <span class="text-category-foreground-l0">{challenge.name}</span>
  <span class="bg-category-background-l0 text-category-foreground-l1">
    {config.name}
  </span>
</div>
```

Each color (red, orange, yellow, green, teal, blue, purple, pink, fuchsia, gray) has five variables defined in `app.css` for both light and dark modes: `background-l0`, `background-l1`, `background-l1-hover`, `foreground-l0`, and `foreground-l1`.
:::

## Adding a custom category

::::steps

### Define the category

Add an entry to `categoryConfigs` with a name, icon component, Iconify name (for exports), and color key:

```ts [lib/utils/categories.ts]
import { IconCpuFilled } from '$lib/icons'

export const categoryConfigs: Record<string, CategoryConfig> = {
  // ... existing
  hardware: { // [!code ++]
    name: 'Hardware', // [!code ++]
    icon: IconCpuFilled, // [!code ++]
    iconName: 'tabler:cpu-filled', // [!code ++]
    color: 'teal', // [!code ++] Use an existing color, or add a new one
  },
}
```

### Register the icon

In `lib/icons/index.ts`, export the icon:

```ts [lib/icons/index.ts]
export { default as IconAlertCircleFilled } from '@iconify-svelte/tabler/alert-circle-filled'
export { default as IconCameraFilled } from '@iconify-svelte/tabler/camera-filled'
export { default as IconCpuFilled } from '@iconify-svelte/tabler/cpu-filled' // [!code ++] // [!code focus]
export { default as IconPhotoFilled } from '@iconify-svelte/tabler/photo-filled'
export { default as IconAlertTriangleFilled } from '@iconify-svelte/tabler/alert-triangle-filled'
```

### Set the display order (optional)

Add your category to `categoryOrder{:ts}` to control its position in the challenge list:

```ts [lib/utils/categories.ts]
export const categoryOrder = [
  'sanity', 'pwn', 'reverse', 'crypto', 'forensics', 
  'blockchain', 'web', 'misc', 'ppc', 'koth', 'osint',
  'hardware', // [!code ++]
]
```

Categories not in this list appear alphabetically after listed ones.

### Add color variables (if using a new color)

If your category uses a color not already in the palette, add variables to `app.css` following the existing pattern. Define `--background-{color}-l0{:css}`, `--background-{color}-l1{:css}`, `--background-{color}-l1-hover{:css}`, `--foreground-{color}-l0{:css}`, and `--foreground-{color}-l1{:css}` for both `:root` and `[data-theme='dark']{:css}`.

::::

## Utility functions

The following utility functions are available to help with category configuration:

| Function | Purpose |
|----------|---------|
| `getCategoryConfig(category){:ts}` | Returns `{ name, icon, iconName, color }{:ts}` for a category key |
| `getCategoryKeyOrAlias(category){:ts}` | Resolves aliases to canonical keys (`'rev'` → `'reverse'`) |
| `getCategoryStyle(color){:ts}` | Generates inline CSS for category variable binding |
| `getCategoryOrder(category){:ts}` | Returns sort index (`-1` if unlisted) |
