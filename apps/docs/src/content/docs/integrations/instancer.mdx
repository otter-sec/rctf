title: Instancer

The instancer integration provides per-team challenge instances — isolated environments that are created, managed, and cleaned up automatically. This is essential for challenges where teams need their own running service (e.g., pwn, web exploitation).

## Configuration

Set up the instancer provider in your configuration:

```yaml title="rctf.d/instancer.yaml"
instancerProvider:
  name: instancer/docker-instancer
  options:
    apiUrl: http://localhost:8000
    authToken: your-secret-token
```

## Providers

<Tabs items={['instancer/docker-instancer', 'instancer/k8s-instancer']}>
<Tab>

Uses [tiny-instancer](https://github.com/otter-sec/tiny-instancer) or a compatible Docker-based instancer API.

```yaml
instancerProvider:
  name: instancer/docker-instancer
  options:
    apiUrl: http://localhost:8000
    authToken: your-auth-token
```

| Option             | Environment Variable             | Description          |
| ------------------ | -------------------------------- | -------------------- |
| `[31mapiUrl[0m{:ansi}`    | `TINY_INSTANCER_API_URL{:sh}`    | Instancer API URL    |
| `[31mauthToken[0m{:ansi}` | `TINY_INSTANCER_AUTH_TOKEN{:sh}` | Authentication token |

</Tab>
<Tab>

Uses Kubernetes for instance management, creating ChallengeInstance custom resources.

```yaml
instancerProvider:
  name: instancer/k8s-instancer
  options:
    apiUrl: https://k8s-instancer.example.com
    authToken: your-auth-token
    caCertificate: |
      -----BEGIN CERTIFICATE-----
      ...
      -----END CERTIFICATE-----
```

| Option                 | Environment Variable                | Description                   |
| ---------------------- | ----------------------------------- | ----------------------------- |
| `[31mapiUrl[0m{:ansi}`        | `K8S_INSTANCER_API_URL{:sh}`        | K8s instancer API URL         |
| `[31mauthToken[0m{:ansi}`     | `K8S_INSTANCER_AUTH_TOKEN{:sh}`     | Authentication token          |
| `[31mcaCertificate[0m{:ansi}` | `K8S_INSTANCER_CA_CERTIFICATE{:sh}` | TLS CA certificate (optional) |

</Tab>
</Tabs>

## Challenge configuration

Each challenge that uses the instancer needs an `[31minstancerConfig[0m{:ansi}` in its challenge data. This is set through the admin challenge editor.

Key fields in `[31minstancerConfig[0m{:ansi}`:

| Field                        | Description                                                                |
| ---------------------------- | -------------------------------------------------------------------------- |
| `[31mconfig[0m{:ansi}`              | Provider-specific instance configuration (Docker compose-like or K8s spec) |
| `[31mexpose[0m{:ansi}`              | Array of endpoints to expose to participants                               |
| `[31mtimeoutMilliseconds[0m{:ansi}` | Instance lifetime before automatic cleanup                                 |
| `[31mextendable[0m{:ansi}`          | Whether participants can extend the instance lifetime                      |

### Endpoint exposure

The `[31mexpose[0m{:ansi}` array controls which ports and services are visible to participants:

| Field                  | Description                                       |
| ---------------------- | ------------------------------------------------- |
| `[31mkind[0m{:ansi}`          | Endpoint type (e.g., `[32mtcp[0m{:ansi}`, `[32mhttp[0m{:ansi}`) |
| `[31mhostPrefix[0m{:ansi}`    | Prefix for the endpoint URL                       |
| `[31mcontainerName[0m{:ansi}` | Name of the container exposing the port           |
| `[31mcontainerPort[0m{:ansi}` | Port number inside the container                  |
| `[31mshouldDisplay[0m{:ansi}` | Whether this endpoint is shown to participants    |
| `[31mtitle[0m{:ansi}`         | Display label for the endpoint                    |

Only endpoints with `[31mshouldDisplay[0m{:ansi}` set to `true{:ts}` are shown to participants. This allows internal service ports to remain hidden.

## Instance lifecycle

Participants can manage their instances through the platform:

| Action | Description                                           |
| ------ | ----------------------------------------------------- |
| Create | Start a new instance for the challenge                |
| Status | Check instance status and connection endpoints        |
| Extend | Extend instance lifetime (if the challenge allows it) |
| Delete | Manually stop an instance                             |

The create and extend actions can be protected with captcha (see [Captcha Providers](/providers/captcha)). Use `[32minstancerStart[0m{:ansi}` and `[32minstancerExtend[0m{:ansi}` in `[31mprotectedEndpoints[0m{:ansi}`.

## Instancer config schema

Admins can retrieve the provider-specific configuration JSON schema through the admin panel. This is useful for understanding the expected format when configuring challenge instances.
