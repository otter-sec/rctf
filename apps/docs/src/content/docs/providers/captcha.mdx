Captcha providers protect sensitive endpoints from automated abuse. rCTF supports three captcha services.

## Configuration

Captcha configuration has two parts: the provider (which service to use) and the protected endpoints (which actions require verification).

```yaml
captcha:
  provider:
    name: captcha/turnstile
    options:
      siteKey: your-site-key
      secretKey: your-secret-key
  protectedEndpoints:
    - register
    - recover
```

## Protected endpoints

The `[31mprotectedEndpoints[0m{:ansi}` array controls which actions require captcha verification:

| Action                          | Description                             |
| ------------------------------- | --------------------------------------- |
| `[32mregister[0m{:ansi}`        | New account registration                |
| `[32mrecover[0m{:ansi}`         | Account recovery via email              |
| `[32msetEmail[0m{:ansi}`        | Changing account email                  |
| `[32minstancerStart[0m{:ansi}`  | Starting a challenge instance           |
| `[32minstancerExtend[0m{:ansi}` | Extending a challenge instance lifetime |
| `[32mavatarUpload[0m{:ansi}`    | Uploading a team avatar                 |
| `[32madminBotSubmit[0m{:ansi}`  | Submitting a job to the admin bot       |

Only the actions listed in `[31mprotectedEndpoints[0m{:ansi}` will require captcha. Unlisted actions proceed without verification.

## Providers

<Tabs items={['captcha/recaptcha', 'captcha/hcaptcha', 'captcha/turnstile']}>
<Tab>

Google reCAPTCHA (v2 or v3).

```yaml
captcha:
  provider:
    name: captcha/recaptcha
    options:
      siteKey: your-site-key
      secretKey: your-secret-key
```

| Option                    | Environment Variable             | Description          |
| ------------------------- | -------------------------------- | -------------------- |
| `[31msiteKey[0m{:ansi}`   | `RCTF_RECAPTCHA_SITE_KEY{:sh}`   | reCAPTCHA site key   |
| `[31msecretKey[0m{:ansi}` | `RCTF_RECAPTCHA_SECRET_KEY{:sh}` | reCAPTCHA secret key |

</Tab>

## Migrating from v1

If you are using the legacy `[31mrecaptcha[0m{:ansi}` top-level config from v1, it is automatically converted to the new format at startup. However, migrating is recommended:

<Tabs items={['v1 (deprecated)', 'v2 (recommended)']}>
<Tab>

```yaml
recaptcha:
  siteKey: your-site-key
  secretKey: your-secret-key
  protectedActions:
    - register
```

</Tab>
<Tab>

```yaml
captcha:
  provider:
    name: captcha/recaptcha
    options:
      siteKey: your-site-key
      secretKey: your-secret-key
  protectedEndpoints:
    - register
```

</Tab>
</Tabs>
