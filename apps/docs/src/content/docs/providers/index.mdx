rCTF uses a pluggable provider system that allows you to choose between different implementations for key platform features. Each provider is configured with a `[31mname[0m{:ansi}` and an `[31moptions[0m{:ansi}` object in your [configuration](/configuration/).

## Provider format

All providers follow the same configuration pattern:

```yaml
providerField:
  name: provider-type/provider-name
  options:
    key: value
```

The `[31mname[0m{:ansi}` identifies which implementation to load, and `[31moptions[0m{:ansi}` is passed directly to the provider's constructor. Most providers also support environment variable fallbacks for their options.
