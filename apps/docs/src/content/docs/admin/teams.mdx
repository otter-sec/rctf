title: Teams
description: Managing teams and users in rCTF.
---

rCTF uses a team-based model where each team shares a single login token. The admin panel provides tools for viewing, searching, and managing teams.

## Listing and searching teams

The admin panel lets you browse all registered teams with their scores. You can paginate through teams and search by name.

## Team tokens

Each team authenticates with a shared team token. Admins can generate a new login token for any non-privileged team. This is useful for helping teams that have lost access to their account.

## Divisions

Teams are assigned to a division at registration. Divisions are configured in the `divisions{:yaml}` config option:

```yaml
divisions:
  open: Open
  student: Student
defaultDivision: open
```

Division assignment can be restricted by email using [division ACLs](/configuration#division-acls).

## Team members

When `userMembers{:yaml}` is enabled in the configuration (default: `true{:ts}`), teams can add members by email. This is informational - members share the same team token.

| Setting       | Default | Description                 |
| ------------- | ------- | --------------------------- |
| `userMembers{:yaml}` | `true{:ts}`  | Enable team members feature |
| `maxMembers{:yaml}`  | `50{:ts}`    | Maximum members per team    |

## Account requirements

Every user account must have at least one authentication method:

- **Email** - Used for registration, recovery, and email-based features
- **CTFtime ID** - Used for CTFtime OAuth authentication

A user cannot remove their email if they have no CTFtime ID linked, and vice versa.
