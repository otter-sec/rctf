Submissions (solves) are recorded when a team successfully submits the correct flag for a challenge. Admins can view and manage solves through the admin panel, and can review submission IP logs for both flag submissions and admin bot submissions.

## Viewing solves

Admins can view solves for any challenge, including the team name, solve timestamp, and ranking information.

## Deleting solves

Admins with the `challsSolveWrite` permission can delete individual solves. This is useful for handling cheating or accidental solves.

<Callout variant="warning">

Deleting a solve triggers an automatic leaderboard recalculation. The team's score and rank will be updated to reflect the removal.

</Callout>
## Submission tracking

Each solve record includes the IP address of the submitter. The admin panel also includes a submission log table that records IPs for flag attempts and admin bot submissions, with sorting by time, challenge, team, IP, submission type, and result. This can be used for investigating suspicious activity such as flag sharing between teams.

## First bloods

The first three teams to solve a challenge are tracked as "bloods":

- **First blood** (index 0)
- **Second blood** (index 1)
- **Third blood** (index 2)

Blood status is determined by the solve timestamp. If a [blood bot](/integrations/bloodbot) is configured, announcements are sent automatically when these milestones are reached.
