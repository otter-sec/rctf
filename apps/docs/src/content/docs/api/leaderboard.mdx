title: Leaderboard
description: API reference for leaderboard standings, graph data, challenge metadata, search, and pagination.
---

Leaderboard routes read cached standings produced by the leaderboard worker. The worker calculates current challenge scores, team scores, global ranks, division ranks, first bloods, and score graph samples.

## Routes

| Method | Path | Version | Auth | Gate |
| --- | --- | --- | --- | --- |
| `<cyan>GET</cyan>` | `<black>/api/v2/leaderboard/now</black>` | V2 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `<cyan>GET</cyan>` | `<black>/api/v2/leaderboard/with-graph</black>` | V2 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `<cyan>GET</cyan>` | `<black>/api/v2/leaderboard/graph</black>` | V2 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `<cyan>GET</cyan>` | `<black>/api/v2/leaderboard/challs</black>` | V2 | Public + optional bypass (`challsRead{:ts}`) | Started |
| `<cyan>GET</cyan>` | `<black>/api/v1/leaderboard/now</black>` | V1 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `<cyan>GET</cyan>` | `<black>/api/v1/leaderboard/graph</black>` | V1 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |

Routes accept an optional auth token. When the token carries the listed permission bit, it bypasses the start-time gate. `<black>/api/v2/leaderboard/challs</black>` uses `challsRead{:ts}` instead of `leaderboardRead{:ts}`.

## Query parameters

`<black>/now</black>`, `<black>/with-graph</black>`, and `<black>/graph</black>` use query-string pagination. The shared parameters:

| Query | Routes | Required | Notes |
| --- | --- | --- | --- |
| `limit` | All list and graph routes | Yes | Integer greater than or equal to `1`. Config enforces the maximum. |
| `offset` | V2 `<black>/now</black>`, V2 `<black>/with-graph</black>`, V2 `<black>/graph</black>`, V1 `<black>/now</black>` | Yes | Integer greater than or equal to `0`. Config enforces the maximum. |
| `division` | All list and graph routes | No | Filters standings to a configured division. |
| `search` | V2 `<black>/now</black>`, V2 `<black>/with-graph</black>` only | No | Fuzzy team-name search. Must be 2 to 100 characters. |

V1 `<black>/api/v1/leaderboard/graph</black>` accepts `limit` and optional `division`, but it does not accept `offset`.

Search requests are rate-limited per IP address (burst `3`, refill window `3000` ms — captcha-free but rate-limited) and return `<red>429 badRateLimit</red>` with `data.timeLeft` when the bucket is exhausted.

## V2 routes

### `<cyan>GET</cyan> <black>/api/v2/leaderboard/now</black>`

Returns the current standings page.

| Property       | Value                                                                        |
| -------------- | ---------------------------------------------------------------------------- |
| Auth           | Public + optional bypass (`leaderboardRead{:ts}`)                            |
| Gate           | Started                                                                      |
| Permissions    | —                                                                            |
| Captcha action | —                                                                            |
| Rate limit     | Search bucket per IP (burst `3`, refill `3000` ms) when `search` is provided |

**Query**

See [Query parameters](#query-parameters). Accepts `limit`, `offset`, optional `division`, and optional `search`.

**Response**

`<green>200 goodLeaderboard</green>`:

```json
{
  "kind": "goodLeaderboard",
  "message": "The leaderboard was successfully retrieved.",
  "data": {
    "total": 37,
    "leaderboard": [
      {
        "id": "team-id",
        "name": "maple bacon",
        "score": 1200,
        "avatarUrl": null,
        "countryCode": "US",
        "statusText": null,
        "solves": [{ "id": "challenge-id", "solveTime": 1710000000000 }],
        "division": "open",
        "divisionPlace": 1,
        "globalPlace": 3
      }
    ]
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `total` | `number{:ts}` | Total matching teams before pagination. |
| `leaderboard[]` | `LeaderboardEntryV2[]{:ts}` | Paginated teams. See [`LeaderboardEntryV2`](/api/responses/). |
| `globalPlace` | `number \| null{:ts}` | Global rank. `null{:ts}` when the team is filtered out of global ranking (for example, banned teams). |
| `divisionPlace` | `number{:ts}` | Rank inside the returned division context. |
| `solves[]` | `{ id: string, solveTime: number }[]{:ts}` | Challenge IDs and solve timestamps. |

**Errors**: `<red>400 badBody</red>`, `<red>401 badNotStarted</red>`, `<red>429 badRateLimit</red>`.

### `<cyan>GET</cyan> <black>/api/v2/leaderboard/with-graph</black>`

Returns the same standings payload as `<black>/now</black>`, plus graph samples. Useful for clients that render the standings table and the score graph together.

| Property       | Value                                                                        |
| -------------- | ---------------------------------------------------------------------------- |
| Auth           | Public + optional bypass (`leaderboardRead{:ts}`)                            |
| Gate           | Started                                                                      |
| Permissions    | —                                                                            |
| Captcha action | —                                                                            |
| Rate limit     | Search bucket per IP (burst `3`, refill `3000` ms) when `search` is provided |

**Query**

See [Query parameters](#query-parameters). Accepts `limit`, `offset`, optional `division`, and optional `search`.

**Response**

`<green>200 goodLeaderboardWithGraph</green>`:

```json
{
  "kind": "goodLeaderboardWithGraph",
  "message": "The leaderboard was successfully retrieved.",
  "data": {
    "total": 37,
    "leaderboard": [],
    "graph": [
      {
        "id": "team-id",
        "name": "maple bacon",
        "points": [{ "time": 1710000000000, "score": 500 }]

    ]
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `total` | `number{:ts}` | Total matching teams before pagination. |
| `leaderboard[]` | `LeaderboardEntryV2[]{:ts}` | Paginated teams. See [`LeaderboardEntryV2`](/api/responses/). |
| `graph[]` | `LeaderboardGraphEntry[]{:ts}` | Teams included in the graph sample. See [`LeaderboardGraphEntry`](/api/responses/). |
| `graph[].points[]` | `{ time: number, score: number }[]{:ts}` | Score samples over time. |

**Errors**: `<red>400 badBody</red>`, `<red>401 badNotStarted</red>`, `<red>429 badRateLimit</red>`.

Returns only graph data.

| Property       | Value                                             |
| -------------- | ------------------------------------------------- |

| Gate           | Started                                           |
| Permissions    | —                                                 |
| Captcha action | —                                                 |
| Rate limit     | —                                                 |

**Query**

See [Query parameters](#query-parameters). Accepts `limit`, `offset`, and optional `division`. Does not accept `search`.

**Response**

`<green>200 goodLeaderboardGraph</green>`:

```json
{
  "kind": "goodLeaderboardGraph",
  "message": "The leaderboard graph was successfully retrieved.",
  "data": {
    "graph": [
      {
        "id": "team-id",
        "name": "maple bacon",
```

| Field | Type | Notes |
| --- | --- | --- |

## Ranking rules

The leaderboard worker sorts teams by score descending. Ties are broken in this order:

1. Most recent solve of a challenge marked `tiebreakEligible` (`last_tiebreak_solve_at`) — earlier is better.
2. Absolute last solve time — earlier is better.

This chain provides a stable tiebreak input when both scores and regular last-solve times match.

## Banned teams

Banned teams are excluded from leaderboard output. Their cached rank fields are cleared until they are unbanned and the leaderboard is recalculated. When a banned team does appear in any response, `globalPlace` is `null{:ts}`.
