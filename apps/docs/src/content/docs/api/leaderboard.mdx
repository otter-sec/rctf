Leaderboard routes read cached standings produced by the leaderboard worker. The worker calculates current challenge scores, team scores, global ranks, division ranks, first bloods, and score graph samples.

## Routes

| Method | Path | Version | Auth | Gate |
| --- | --- | --- | --- | --- |
| `[36mGET[0m{:ansi}` | `[2m/api/[22mv2[2m/[22mleaderboard[2m/[22mnow{:ansi}` | V2 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `[36mGET[0m{:ansi}` | `[2m/api/[22mv2[2m/[22mleaderboard[2m/[22mwith-graph{:ansi}` | V2 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `[36mGET[0m{:ansi}` | `[2m/api/[22mv2[2m/[22mleaderboard[2m/[22mgraph{:ansi}` | V2 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `[36mGET[0m{:ansi}` | `[2m/api/[22mv2[2m/[22mleaderboard[2m/[22mchalls{:ansi}` | V2 | Public + optional bypass (`challsRead{:ts}`) | Started |
| `[36mGET[0m{:ansi}` | `[2m/api/[22mv1[2m/[22mleaderboard[2m/[22mnow{:ansi}` | V1 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `[36mGET[0m{:ansi}` | `[2m/api/[22mv1[2m/[22mleaderboard[2m/[22mgraph{:ansi}` | V1 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |

Routes accept an optional auth token. When the token carries the listed permission bit, it bypasses the start-time gate. `[2m/api/[22mv2[2m/[22mleaderboard[2m/[22mchalls{:ansi}` uses `challsRead{:ts}` instead of `leaderboardRead{:ts}`.

## Query parameters

`[2m/[22mnow{:ansi}`, `[2m/[22mwith-graph{:ansi}`, and `[2m/[22mgraph{:ansi}` use query-string pagination. The shared parameters:

| Query | Routes | Required | Notes |
| --- | --- | --- | --- |
| `limit` | All list and graph routes | Yes | Integer greater than or equal to `1`. Config enforces the maximum. |
| `offset` | V2 `[2m/[22mnow{:ansi}`, V2 `[2m/[22mwith-graph{:ansi}`, V2 `[2m/[22mgraph{:ansi}`, V1 `[2m/[22mnow{:ansi}` | Yes | Integer greater than or equal to `0`. Config enforces the maximum. |
| `division` | All list and graph routes | No | Filters standings to a configured division. |
| `search` | V2 `[2m/[22mnow{:ansi}`, V2 `[2m/[22mwith-graph{:ansi}` only | No | Fuzzy team-name search. Must be 2 to 100 characters. |

V1 `[2m/api/[22mv1[2m/[22mleaderboard[2m/[22mgraph{:ansi}` accepts `limit` and optional `division`, but it does not accept `offset`.

Search requests are rate-limited per IP address (burst `3`, refill window `3000` ms — captcha-free but rate-limited) and return `[2;31m429 [22mbadRateLimit[0m{:ansi}` with `data.timeLeft` when the bucket is exhausted.

## Ranking rules

The leaderboard worker sorts teams by score descending. Ties are broken in this order:

1. Most recent solve of a challenge marked `tiebreakEligible` (`last_tiebreak_solve_at`) — earlier is better.
2. Absolute last solve time — earlier is better.

This chain provides a stable tiebreak input when both scores and regular last-solve times match.

## Banned teams

Banned teams are excluded from leaderboard output. Their cached rank fields are cleared until they are unbanned and the leaderboard is recalculated. When a banned team does appear in any response, `globalPlace` is `null{:ts}`.
