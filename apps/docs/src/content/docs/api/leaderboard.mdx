---
title: Leaderboard
description: API reference for leaderboard standings, graph data, challenge metadata, search, and pagination.
---

Leaderboard routes read cached standings produced by the leaderboard worker. The worker calculates current challenge scores, team scores, global ranks, division ranks, first bloods, and score graph samples.

## Routes

| Method | Path | Version | Auth | Query |
| --- | --- | --- | --- | --- |
| `GET{:http}` | `/api/v2/leaderboard/now` | V2 | Public, optional admin bypass | Required |
| `GET{:http}` | `/api/v2/leaderboard/with-graph` | V2 | Public, optional admin bypass | Required |
| `GET{:http}` | `/api/v2/leaderboard/graph` | V2 | Public, optional admin bypass | Required |
| `GET{:http}` | `/api/v2/leaderboard/challs` | V2 | Public, optional admin bypass | None |
| `GET{:http}` | `/api/v1/leaderboard/now` | V1 | Public, optional admin bypass | Required |
| `GET{:http}` | `/api/v1/leaderboard/graph` | V1 | Public, optional admin bypass | Required |

Routes with leaderboard start bypass accept an auth token with `leaderboardRead`. `/api/v2/leaderboard/challs` uses `challsRead` for its start bypass.

## Query parameters

`/now`, `/with-graph`, and `/graph` use query-string pagination:

| Query | Routes | Required | Notes |
| --- | --- | --- | --- |
| `limit` | All leaderboard list and graph routes | Yes | Integer greater than or equal to `1`. Config enforces the maximum. |
| `offset` | V2 `/now`, V2 `/with-graph`, V2 `/graph`, V1 `/now` | Yes | Integer greater than or equal to `0`. Config enforces the maximum. |
| `division` | All leaderboard list and graph routes | No | Filters standings to a configured division. |
| `search` | V2 `/now`, V2 `/with-graph` | No | Fuzzy team-name search. Must be 2 to 100 characters. |

V1 `/api/v1/leaderboard/graph` accepts `limit` and optional `division`, but it does not accept `offset`.

Search requests are rate-limited per IP address and return `badRateLimit` with `data.timeLeft` when the search bucket is exhausted.

## `GET /api/v2/leaderboard/now`

Returns the current standings:

```json
{
  "total": 37,
  "leaderboard": [
    {
      "id": "team-id",
      "name": "maple bacon",
      "score": 1200,
      "avatarUrl": null,
      "countryCode": "US",
      "statusText": null,
      "solves": [{ "id": "challenge-id", "solveTime": 1710000000000 }],
      "division": "open",
      "divisionPlace": 1,
      "globalPlace": 3
    }
  ]
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `total` | `number` | Total matching teams before pagination. |
| `leaderboard[]` | `LeaderboardEntryV2[]` | Paginated teams. |
| `globalPlace` | `number \| null` | Global rank. It can be `null` when the team is filtered out of global ranking, such as banned teams. |
| `divisionPlace` | `number` | Rank inside the returned division context. |
| `solves[]` | `{ id: string, solveTime: number }[]` | Challenge IDs and solve timestamps. |

## `GET /api/v2/leaderboard/with-graph`

Returns the same standing payload as `/now`, plus graph samples:

```json
{
  "total": 37,
  "leaderboard": [],
  "graph": [
    {
      "id": "team-id",
      "name": "maple bacon",
      "points": [
        { "time": 1710000000000, "score": 500 }
      ]
    }
  ]
}
```

This route is useful for clients that render the standings table and score graph together.

## `GET /api/v2/leaderboard/graph`

Returns only graph data:

| Field | Type | Notes |
| --- | --- | --- |
| `graph[]` | `LeaderboardGraphEntry[]` | Teams included in the graph sample. |
| `graph[].points[]` | `{ time: number, score: number }[]` | Score samples over time. |

The graph uses the worker's cached sample set. `leaderboard.graphMaxTeams` controls how many teams are kept for graphing, and `leaderboard.graphSampleTime` controls sample cadence.

## `GET /api/v2/leaderboard/challs`

Returns challenge metadata used by leaderboard visualizations:

```json
{
  "challenges": {
    "challenge-id": {
      "name": "web demo",
      "category": "web",
      "solves": 12,
      "points": 443,
      "sortWeight": null,
      "firstSolvers": [{ "id": "team-id" }]
    }
  }
}
```

`firstSolvers` contains up to the first three team IDs for each challenge.

## V1 leaderboard data

V1 standings are smaller:

```json
{
  "total": 37,
  "leaderboard": [
    {
      "id": "team-id",
      "name": "maple bacon",
      "score": 1200
    }
  ]
}
```

V1 graph data uses the same `goodLeaderboardGraph` shape as v2.

## Ranking rules

The leaderboard worker sorts teams by score descending. Ties are broken by the time the team reached its score. The worker also tracks `last_tiebreak_solve_at` for challenges marked as `tiebreakEligible`, which provides an additional stable tiebreak input when scores and regular last solve times match.

Banned teams are excluded from leaderboard output and have cached rank fields cleared until they are unbanned and the leaderboard is recalculated.
