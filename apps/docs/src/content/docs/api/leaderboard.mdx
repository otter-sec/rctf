## Scoring computation

The leaderboard is computed by a background worker that runs at a configurable interval (default 30 seconds, controlled by `leaderboard.updateInterval`). The worker:

1. Collects all solves and computes each challenge's current point value using the [scoring provider](/providers/scores)
2. Sums each team's points across all solved challenges
3. Ranks teams within each division and globally
4. Caches the results in the database and Redis

Certain operations - such as challenge updates, challenge deletions, and solve deletions - trigger an immediate recalculation outside the normal interval.

<Callout variant="note">

If the worker encounters three consecutive failures, it resets its internal calculator state and starts fresh. This provides automatic recovery from transient errors.

</Callout>
## Tiebreaking

When teams have equal scores, the following rules determine their ranking:

1. **Score** (descending) - Higher score ranks higher
2. **Last solve time** - Among teams with equal scores, the team that reached that score first ranks higher (earlier last solve wins)
3. **Last tiebreak-eligible solve time** - Further tiebreaking considers only solves from challenges marked as `tiebreakEligible`

This means that solving a challenge earlier gives a competitive advantage when scores are tied.

## Divisions

Teams are assigned to a division at registration. The leaderboard supports filtering by division, returning only teams in a specific division with division-specific rankings. Global rankings across all divisions are always available as well.

## First bloods

The leaderboard tracks the first three solvers for each challenge. These "bloods" are surfaced in challenge leaderboard data and can trigger notifications via the [blood bot](/integrations/bloodbot).

## Search

The leaderboard supports fuzzy team name search. Searches require at least 2 characters and are case-insensitive. Search requests have a separate rate limit (3 per 3 seconds per IP).

## Score graph

The platform captures periodic snapshots of team scores over time to power the score graph visualization. Data points are sampled at intervals controlled by `leaderboard.graphSampleTime` (default 30 minutes). The graph shows the top N teams (controlled by `leaderboard.graphMaxTeams`, default 10).

## Pagination

Leaderboard queries support `limit` and `offset` pagination. Maximum values are controlled by the `leaderboard.maxLimit` (default 100) and `leaderboard.maxOffset` configuration options.
