title: Leaderboard
description: API reference for leaderboard standings, graph data, challenge metadata, search, and pagination.

Leaderboard routes read cached standings produced by the leaderboard worker. The worker calculates current challenge scores, team scores, global ranks, division ranks, first bloods, and score graph samples.

## Routes

| Method | Path | Version | Auth | Gate |
| --- | --- | --- | --- | --- |
| `GET{:http}` | `/api/v2/leaderboard/now` | V2 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `GET{:http}` | `/api/v2/leaderboard/with-graph` | V2 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `GET{:http}` | `/api/v2/leaderboard/graph` | V2 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `GET{:http}` | `/api/v2/leaderboard/challs` | V2 | Public + optional bypass (`challsRead{:ts}`) | Started |
| `GET{:http}` | `/api/v1/leaderboard/now` | V1 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |
| `GET{:http}` | `/api/v1/leaderboard/graph` | V1 | Public + optional bypass (`leaderboardRead{:ts}`) | Started |

Routes accept an optional auth token. When the token carries the listed permission bit, it bypasses the start-time gate. `/api/v2/leaderboard/challs` uses `challsRead{:ts}` instead of `leaderboardRead{:ts}`.

## Query parameters

`/now`, `/with-graph`, and `/graph` use query-string pagination. The shared parameters:

| Query | Routes | Required | Notes |
| --- | --- | --- | --- |
| `limit` | All list and graph routes | Yes | Integer greater than or equal to `1`. Config enforces the maximum. |
| `offset` | V2 `/now`, V2 `/with-graph`, V2 `/graph`, V1 `/now` | Yes | Integer greater than or equal to `0`. Config enforces the maximum. |
| `division` | All list and graph routes | No | Filters standings to a configured division. |
| `search` | V2 `/now`, V2 `/with-graph` only | No | Fuzzy team-name search. Must be 2 to 100 characters. |

V1 `/api/v1/leaderboard/graph` accepts `limit` and optional `division`, but it does not accept `offset`.

Search requests are rate-limited per IP address (burst `3`, refill window `3000` ms ‚Äî captcha-free but rate-limited) and return `badRateLimit` with `data.timeLeft` when the bucket is exhausted.

## V2 routes

### `GET /api/v2/leaderboard/now`

Returns the current standings page.

| Property | Value |
| --- | --- |
| Auth | Public + optional bypass (`leaderboardRead{:ts}`) |
| Gate | Started |
| Permissions | ‚Äî |
| Captcha action | ‚Äî |
| Rate limit | Search bucket per IP (burst `3`, refill `3000` ms) when `search` is provided |

**Query**

See [Query parameters](#query-parameters). Accepts `limit`, `offset`, optional `division`, and optional `search`.

**Response**

`goodLeaderboard` (200):

```json
{
  "kind": "goodLeaderboard",
  "message": "The leaderboard was successfully retrieved.",
  "data": {
    "total": 37,
    "leaderboard": [
      {
        "id": "team-id",
        "name": "maple bacon",
        "score": 1200,
        "avatarUrl": null,
        "countryCode": "US",
        "statusText": null,
        "solves": [{ "id": "challenge-id", "solveTime": 1710000000000 }],
        "division": "open",
        "divisionPlace": 1,
        "globalPlace": 3
      }
    ]
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `total` | `number{:ts}` | Total matching teams before pagination. |
| `leaderboard[]` | `LeaderboardEntryV2[]{:ts}` | Paginated teams. See [`LeaderboardEntryV2`](/api/responses). |
| `globalPlace` | `number \| null{:ts}` | Global rank. `null{:ts}` when the team is filtered out of global ranking (for example, banned teams). |
| `divisionPlace` | `number{:ts}` | Rank inside the returned division context. |
| `solves[]` | `{ id: string, solveTime: number }[]{:ts}` | Challenge IDs and solve timestamps. |

**Errors**: `badBody`, `badNotStarted`, `badRateLimit`.

## Ranking rules

The leaderboard worker sorts teams by score descending. Ties are broken in this order:

1. Most recent solve of a challenge marked `tiebreakEligible` (`last_tiebreak_solve_at`) ‚Äî earlier is better.
2. Absolute last solve time ‚Äî earlier is better.

This chain provides a stable tiebreak input when both scores and regular last-solve times match.

## Banned teams

Banned teams are excluded from leaderboard output. Their cached rank fields are cleared until they are unbanned and the leaderboard is recalculated. When a banned team does appear in any response, `globalPlace` is `null{:ts}`.
