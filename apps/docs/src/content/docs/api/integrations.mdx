## Analytics script

### `[36mGET[0m [2m/api/[22mv2[2m/[22mintegrations[2m/[22manalytics[2m/[22mscript{:ansi}`

Returns the active analytics provider's JavaScript body. This route is not part of `@rctf/types`.

| Property       | Value  |
| -------------- | ------ |
| Auth           | Public |
| Gate           | None   |
| Permissions    | —      |
| Captcha action | —      |
| Rate limit     | —      |

**Response**

The route returns the provider's JavaScript body or a plain text error. It is not wrapped in the rCTF envelope. The API caches the script in memory for one hour. When no analytics provider is configured the route returns `404`. Upstream fetch failures return `502`.

## Instancer lifecycle

All four methods share the path `[2m/api/[22mv2[2m/[22mintegrations[2m/[22mchalls[2m/[22m:id[2m/[22minstance{:ansi}` and the same path params. The route returns `[2;31m404 [22mbadEndpoint[0m{:ansi}` when no instancer provider is configured or the challenge has no instancer config. Provider failures return `[2;31m400 [22mbadInstancerError[0m{:ansi}` with `data.message`. Admins with `challsRead` can bypass the start-time gate.

### `[32mPOST[0m [2m/api/[22mv2[2m/[22mintegrations[2m/[22mchalls[2m/[22m:id[2m/[22madmin-bot{:ansi}`

Submits a participant admin bot job. Participants supply values for the configured input names.

| Property       | Value                                                       |
| -------------- | ----------------------------------------------------------- |
| Auth           | Required                                                    |
| Gate           | Started                                                     |
| Permissions    | `challsRead{:ts}` bypasses the start-time gate.             |
| Captcha action | `adminBotSubmit{:ts}` (when configured)                     |
| Rate limit     | Burst `1`, refill window `10000` ms per user and challenge. |

**Path params**

| Field | Type          | Required | Notes         |
| ----- | ------------- | -------- | ------------- |
| `id`  | `string{:ts}` | Yes      | Challenge ID. |

**Request body**

```json
{
  "inputs": {
    "url": "https://challenge.example.com/"
  },
  "captchaCode": "optional-captcha-code"
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `inputs` | `Record<string, string>{:ts}` | Yes | Values for the configured input names. Input names may be up to `256` characters. Input values may be up to `1024` characters. Each value is validated against the regex rule stored in challenge config. |
| `captchaCode` | `string{:ts}` | No | Checked only when captcha protects `adminBotSubmit{:ts}`. |

Only one queued or running job per user per challenge is allowed.

**Response**

`[2;32m200 [22mgoodAdminBotJobSubmitted[0m{:ansi}`:

```json
{
  "kind": "goodAdminBotJobSubmitted",
  "message": "The admin bot job was submitted.",
  "data": {
    "jobId": "job-id"
  }
}
```

| Field        | Type          | Notes                     |
| ------------ | ------------- | ------------------------- |
| `data.jobId` | `string{:ts}` | Submitted job identifier. |

The route returns `[2;31m400 [22mbadInstancerState[0m{:ansi}` when the challenge requires a running instance and the current user's instance is stopped, missing, or expires before the admin bot timeout.

**Errors**: `[2;31m400 [22mbadBody[0m{:ansi}`, `[2;31m403 [22mbadCaptcha[0m{:ansi}`, `[2;31m429 [22mbadRateLimit[0m{:ansi}`, `[2;31m400 [22mbadInstancerState[0m{:ansi}`.
