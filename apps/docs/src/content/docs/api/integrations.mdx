## Client config

### `GET /api/v2/integrations/client/config`

Returns the public client configuration consumed by the frontend.

| Property | Value |
| --- | --- |
| Auth | Public |
| Gate | None |
| Permissions | — |
| Captcha action | — |
| Rate limit | — |

**Response**

`goodClientConfig` (200):

```json
{
  "kind": "goodClientConfig",
  "message": "The client config was retrieved.",
  "data": {}
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `meta` | `{ description: string, imageUrl: string }{:ts}` | Page metadata. |
| `homeContent` | `string{:ts}` | Markdown home content. |
| `sponsors` | `Sponsor[]{:ts}` | Sponsor rows. |
| `flagFormatPlaceholder` | `string{:ts}` | Placeholder shown in flag inputs. |
| `analytics` | `ClientProvider \| null{:ts}` | Public analytics provider data. |
| `ctfName` | `string{:ts}` | Display name. |
| `divisions` | `Record<string, string>{:ts}` | Division key to display name map. |
| `defaultDivision` | `string \| null \| undefined{:ts}` | Default division key when configured. |
| `origin` | `string{:ts}` | Public origin URL. |
| `startTime` | `number{:ts}` | Unix milliseconds. |
| `endTime` | `number{:ts}` | Unix milliseconds. |
| `userMembers` | `boolean{:ts}` | Whether team members are enabled. |
| `faviconUrl` | `string \| null{:ts}` | Runtime favicon URL. |
| `logoLightUrl` | `string \| null{:ts}` | Light logo URL. |
| `logoDarkUrl` | `string \| null{:ts}` | Dark logo URL. |
| `emailEnabled` | `boolean{:ts}` | Whether email delivery is configured. |
| `registrationsEnabled` | `boolean \| null{:ts}` | Runtime registration state. |
| `ctftime` | `{ clientId: string } \| null{:ts}` | Public CTFtime OAuth data. |
| `instancerEnabled` | `boolean{:ts}` | Whether an instancer provider is configured. |
| `isArchived` | `boolean{:ts}` | Whether the CTF has ended. |
| `captcha` | `CaptchaClientConfig \| null{:ts}` | Public captcha provider options and protected endpoint map. |

## Analytics script

### `GET /api/v2/integrations/analytics/script`

Returns the active analytics provider's JavaScript body. This route is not part of `@rctf/types`.

| Property | Value |
| --- | --- |
| Auth | Public |
| Gate | None |
| Permissions | — |
| Captcha action | — |
| Rate limit | — |

**Response**

The route returns the provider's JavaScript body or a plain text error. It is not wrapped in the rCTF envelope. The API caches the script in memory for one hour. When no analytics provider is configured the route returns `404`. Upstream fetch failures return `502`.

## CTFtime

### `POST /api/v1/integrations/ctftime/callback`

V1-only. Exchanges a CTFtime OAuth code for a CTFtime auth token plus the team name and ID returned by CTFtime.

| Property | Value |
| --- | --- |
| Auth | Public |
| Gate | None |
| Permissions | — |
| Captcha action | — |
| Rate limit | — |

**Request body**

```json
{
  "ctftimeCode": "<oauth-code>"
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `ctftimeCode` | `string{:ts}` | Yes | OAuth code returned by CTFtime. |

**Response**

`goodCtftimeToken` (200):

```json
{
  "kind": "goodCtftimeToken",
  "message": "The CTFtime token was retrieved.",
  "data": {
    "ctftimeToken": "<ctftime-auth-token>",
    "ctftimeName": "Team Name",
    "ctftimeId": "12345"
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `data.ctftimeToken` | `string{:ts}` | CTFtime auth token usable on V1 login and V2 register/verify flows. |
| `data.ctftimeName` | `string{:ts}` | CTFtime team name. |
| `data.ctftimeId` | `string{:ts}` | CTFtime team ID. |

**Errors**: `badBody`, `badCtftimeCode`, `badEndpoint`.

## Instancer lifecycle

All four methods share the path `/api/v2/integrations/challs/:id/instance` and the same path params. The route returns `badEndpoint` when no instancer provider is configured or the challenge has no instancer config. Provider failures return `badInstancerError` with `data.message`. Admins with `challsRead` can bypass the start-time gate.

### `GET /api/v2/integrations/challs/:id/instance`

Returns the current instance status for the authenticated user.

| Property | Value |
| --- | --- |
| Auth | Required |
| Gate | Started |
| Permissions | `challsRead{:ts}` bypasses the start-time gate. |
| Captcha action | — |
| Rate limit | — |

**Path params**

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `id` | `string{:ts}` | Yes | Challenge ID. |

**Response**

`goodInstanceStatus` (200):

```json
{
  "kind": "goodInstanceStatus",
  "message": "The instance status was retrieved.",
  "data": {
    "status": "running",
    "timeLeftMilliseconds": 1800000,
    "endpoints": []
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `data.status` | `"stopped" \| "running" \| "starting" \| "stopping" \| "errored"{:ts}` | Current lifecycle state. |
| `data.timeLeftMilliseconds` | `number \| null{:ts}` | Remaining time for running instances. |
| `data.endpoints` | `Endpoint[] \| null{:ts}` | Public endpoints in challenge expose order. |

`Endpoint` has `kind`, `host`, `port`, and optional `title`. Endpoint kinds are `tcp`, `tcp-ssl`, `http`, and `https`.

**Errors**: `badEndpoint`, `badInstancerError`.

## Admin bot participant routes

Participant admin bot routes live under `/api/v2/integrations/challs/:id/admin-bot/*`. Admins with `challsRead` can bypass the start-time gate.

Returns the released admin bot challenge source and its file extension.

| Property | Value |
| --- | --- |
| Auth | Public |
| Gate | Started |
| Permissions | `challsRead{:ts}` bypasses the start-time gate. |
| Captcha action | — |
| Rate limit | — |

**Path params**

| Field | Type | Required | Notes |

**Response**

`goodAdminBotConfig` (200):

### `POST /api/v2/integrations/challs/:id/admin-bot`

Submits a participant admin bot job. Participants supply values for the configured input names.

| Property | Value |
| --- | --- |
| Auth | Required |
| Gate | Started |
| Permissions | `challsRead{:ts}` bypasses the start-time gate. |
| Captcha action | `adminBotSubmit{:ts}` (when configured) |
| Rate limit | Burst `1`, refill window `10000` ms per user and challenge. |

**Path params**

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `id` | `string{:ts}` | Yes | Challenge ID. |

**Request body**

```json
{
  "inputs": {
    "url": "https://challenge.example.com/"
  },
  "captchaCode": "optional-captcha-code"
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `inputs` | `Record<string, string>{:ts}` | Yes | Values for the configured input names. Input names may be up to `256` characters. Input values may be up to `1024` characters. Each value is validated against the regex rule stored in challenge config. |
| `captchaCode` | `string{:ts}` | No | Checked only when captcha protects `adminBotSubmit{:ts}`. |

Only one queued or running job per user per challenge is allowed.

**Response**

`goodAdminBotJobSubmitted` (200):

```json
{
  "kind": "goodAdminBotJobSubmitted",
  "message": "The admin bot job was submitted.",
  "data": {
    "jobId": "job-id"
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `data.jobId` | `string{:ts}` | Submitted job identifier. |

The route returns `badInstancerState` when the challenge requires a running instance and the current user's instance is stopped, missing, or expires before the admin bot timeout.

**Errors**: `badBody`, `badCaptcha`, `badRateLimit`, `badInstancerState`.
