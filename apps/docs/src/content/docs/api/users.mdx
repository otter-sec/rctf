User routes expose team profiles and account-level mutations. V2 adds avatar, country, and status fields. V1 keeps team members and CTFtime account mutation routes for compatibility.

## Routes

| Method | Path | Version | Auth | Gate |
| --- | --- | --- | --- | --- |
| `GET{:http}` | `/api/v2/users/:id` | V2 | Public, optional admin bypass | Started |
| `GET{:http}` | `/api/v2/users/me` | V2 | Required | None |
| `PATCH{:http}` | `/api/v2/users/me` | V2 | Required | None |
| `PATCH{:http}` | `/api/v2/users/me/avatar` | V2 | Required | None |
| `PUT{:http}` | `/api/v2/users/me/auth/email` | V2 | Required | None |
| `GET{:http}` | `/api/v1/users/:id` | V1 | Public, optional admin bypass | Started |
| `GET{:http}` | `/api/v1/users/me` | V1 | Required | None |
| `PATCH{:http}` | `/api/v1/users/me` | V1 | Required | None |
| `GET{:http}` | `/api/v1/users/me/members` | V1 | Required | None |
| `POST{:http}` | `/api/v1/users/me/members` | V1 | Required | None |
| `DELETE{:http}` | `/api/v1/users/me/members/:id` | V1 | Required | None |
| `PUT{:http}` | `/api/v1/users/me/auth/email` | V1 | Required | None |
| `DELETE{:http}` | `/api/v1/users/me/auth/email` | V1 | Required | None |
| `PUT{:http}` | `/api/v1/users/me/auth/ctftime` | V1 | Required | None |
| `DELETE{:http}` | `/api/v1/users/me/auth/ctftime` | V1 | Required | None |

`GET /api/v2/users/:id` and `GET /api/v1/users/:id` can be read before the CTF starts by an admin token with `challsRead{:ts}`.

## V2 routes

### `GET /api/v2/users/:id`

Returns the public profile of a team.

| Property | Value |
| --- | --- |
| Auth | Public, optional admin bypass |
| Gate | Started |
| Permissions | `challsRead{:ts}` (start-time bypass only) |
| Captcha action | — |
| Rate limit | — |

**Path params**

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `id` | `string{:ts}` | Yes | Team ID. |

**Response**

`goodUserData` (200):

```json
{
  "kind": "goodUserData",
  "message": "The user's data was successfully retrieved.",
  "data": {
    "name": "maple bacon",
    "ctftimeId": null,
    "division": "open",
    "score": 1337,
    "globalPlace": 1,
    "divisionPlace": 1,
    "solves": [],
    "avatarUrl": null,
    "countryCode": "US",
    "statusText": "solving web"
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `name` | `string{:ts}` | Team display name. |
| `ctftimeId` | `string \| null \| undefined{:ts}` | Linked CTFtime ID when present. |
| `division` | `string{:ts}` | Team division. |
| `score` | `number{:ts}` | Cached current score. |
| `globalPlace` | `number \| null{:ts}` | Cached global rank. |
| `divisionPlace` | `number \| null{:ts}` | Cached division rank. |
| `solves` | `UserSolveV2[]{:ts}` | Solved challenges visible to the route. See [`UserSolveV2`](#usersolvev2). |
| `avatarUrl` | `string \| null{:ts}` | Team avatar URL. |
| `countryCode` | `string \| null{:ts}` | ISO 3166-like region code from `@rctf/util`. |
| `statusText` | `string \| null{:ts}` | Team status text. |

**Errors**: `badUnknownUser`, `badNotStarted`.

### `GET /api/v2/users/me`

Returns the authenticated team's own profile, including account-private fields.

| Property | Value |
| --- | --- |
| Auth | Required |
| Gate | None |
| Permissions | — |
| Captcha action | — |
| Rate limit | — |

**Response**

`goodUserSelfData` (200) returns the public profile fields plus account-private data:

```json
{
  "kind": "goodUserSelfData",
  "message": "The user's own data was successfully retrieved.",
  "data": {
    "name": "maple bacon",
    "ctftimeId": null,
    "division": "open",
    "score": 1337,
    "globalPlace": 1,
    "divisionPlace": 1,
    "solves": [],
    "avatarUrl": null,
    "countryCode": "US",
    "statusText": "solving web",
    "id": "<team-id>",
    "email": "team@example.com",
    "teamToken": "<team-token>",
    "allowedDivisions": ["open"],
    "perms": null
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `id` | `string{:ts}` | Team ID. |
| `email` | `string \| null{:ts}` | Email address when set. |
| `teamToken` | `string{:ts}` | Long-lived team token. Treat it as a credential. |
| `allowedDivisions` | `string[]{:ts}` | Divisions available to the team based on ACLs. |
| `perms` | `number \| null{:ts}` | Admin permission bitmask. |

All other fields match the [public profile](#get-apiv2usersid).

**Errors**: `badToken`.

### `PATCH /api/v2/users/me`

Updates profile fields. Recalculates cached user data and triggers a leaderboard update.

| Property | Value |
| --- | --- |
| Auth | Required |
| Gate | None |
| Permissions | — |
| Captcha action | — |
| Rate limit | Profile name update — burst `3`, refill window `180000` ms per user. |

**Request body**

```json
{
  "name": "new team name",
  "division": "open",
  "countryCode": "US",
  "statusText": "solving web"
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `name` | `string{:ts}` | No | 2–64 printable ASCII characters. Name updates are rate-limited. |
| `division` | `string{:ts}` | No | Must be allowed for the team's current email under division ACLs. |
| `countryCode` | `string \| null{:ts}` | No | Region code or `null{:ts}`. |
| `statusText` | `string \| null{:ts}` | No | String up to 60 characters or `null{:ts}`. |

**Response**

`goodUserUpdate` (200):

```json
{
  "kind": "goodUserUpdate",
  "message": "The user was successfully updated.",
  "data": {
    "user": {
      "name": "new team name",
      "email": "team@example.com",
      "division": "open",
      "avatarUrl": null,
      "countryCode": "US",
      "statusText": "solving web"
    }
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `user.name` | `string{:ts}` | Updated team name. |
| `user.email` | `string \| null{:ts}` | Current email. |
| `user.division` | `string{:ts}` | Current division. |
| `user.avatarUrl` | `string \| null{:ts}` | Current avatar URL. |
| `user.countryCode` | `string \| null{:ts}` | Current country code. |
| `user.statusText` | `string \| null{:ts}` | Current status text. |

**Errors**: `badBody`, `badName`, `badKnownName`, `badDivisionNotAllowed`, `badRateLimit`.

### `PATCH /api/v2/users/me/avatar`

Uploads or removes the team avatar. The route expects `multipart/form-data`.

| Property | Value |
| --- | --- |
| Auth | Required |
| Gate | None |
| Permissions | — |
| Captcha action | `avatarUpload{:ts}` (when configured) |
| Rate limit | Avatar upload — burst `2`, refill window `120000` ms per user. |

**Request**

| Form field | Type | Required | Notes |
| --- | --- | --- | --- |
| `avatar` | file | No | Image file. When omitted and an avatar exists, the current avatar is deleted. |
| `captchaCode` | `string{:ts}` | No | Checked only when captcha protects `avatarUpload{:ts}`. |

The route checks `maxAvatarSize`, resizes the image to 256×256 pixels, converts it to WebP, optionally sends it to the moderation provider, uploads it through the active upload provider, and deletes the previous avatar when replacing it.

**Response**

`goodAvatarUpdated` (200):

```json
{
  "kind": "goodAvatarUpdated",
  "message": "The avatar was successfully updated.",
  "data": {
    "url": "https://uploads.example.com/avatars/<id>.webp"
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `url` | `string \| null{:ts}` | New avatar URL, or `null{:ts}` when the avatar was deleted. |

**Errors**: `badBody`, `badAvatarFile`, `badAvatarFileSize`, `badModerationNotPassed`, `badFilesUpload`, `badCaptcha`, `badRateLimit`.

### `PUT /api/v2/users/me/auth/email`

Updates the account email. When email delivery is configured, the route sends a verification email and returns `goodVerifySent`. When email delivery is not configured, it updates the email immediately and returns `goodEmailSet`.

| Property | Value |
| --- | --- |
| Auth | Required |
| Gate | None |
| Permissions | — |
| Captcha action | `setEmail{:ts}` (when configured) |
| Rate limit | — |

**Request body**

```json
{
  "email": "team@example.com",
  "captchaCode": "optional-captcha-code"
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `email` | `string{:ts}` | Yes | New email address. |
| `captchaCode` | `string{:ts}` | No | Checked only when captcha protects `setEmail{:ts}`. |

The route rejects emails that would move the team outside the division ACL (`badEmailChangeDivision`) and rejects duplicate emails (`badKnownEmail`).

**Response**

Behavior branches on email provider configuration:

| Condition | Success kind | Effect |
| --- | --- | --- |
| Email delivery configured | `goodVerifySent` | Sends a verification email; consume the token via [`POST /api/v2/auth/verify`](/api/auth#post-apiv2authverify). |
| Email delivery not configured | `goodEmailSet` | Updates the email immediately. |

Both kinds return no data.

**Errors**: `badBody`, `badEmail`, `badKnownEmail`, `badEmailChangeDivision`, `badCaptcha`, `badZeroAuth`.

## V1 compatibility

V1 routes remain available for legacy clients. Behavior matches V2 except where noted below. Shared response and error semantics apply unless overridden.

### `GET /api/v1/users/:id`

Differs from V2: response omits `avatarUrl`, `countryCode`, `statusText`, and `bloodIndex` (on each entry in `solves`).

### `GET /api/v1/users/me`

Differs from V2: response omits `avatarUrl`, `countryCode`, and `statusText`.

### `PATCH /api/v1/users/me`

Differs from V2: only accepts `name` and `division`. Shares the profile name update rate limit (burst `3`, refill window `180000` ms per user) with [`PATCH /api/v2/users/me`](#patch-apiv2usersme).

### `PUT /api/v1/users/me/auth/email`

Differs from V2: V1 request body uses the V1 captcha field name (`recaptchaCode`). Response semantics match V2 — `goodVerifySent` when email delivery is configured, `goodEmailSet` otherwise. Same division-ACL and duplicate-email rejections apply.

### `DELETE /api/v1/users/me/auth/email`

V1-only. Removes the account email. Preserves the `badZeroAuth` invariant: a team must keep at least one of email or CTFtime auth.

| Property | Value |
| --- | --- |
| Auth | Required |
| Gate | None |
| Permissions | — |
| Captcha action | — |
| Rate limit | — |

**Response**

`goodEmailRemoved` (no data).

**Errors**: `badEmailNoExists`, `badZeroAuth`.

### `PUT /api/v1/users/me/auth/ctftime`

### `DELETE /api/v1/users/me/auth/ctftime`

V1-only. Unlinks the account's CTFtime auth. Removing the last remaining auth method returns `badZeroAuth`, which preserves the invariant that every team has at least one of email or CTFtime auth.

**Response**

**Errors**: `badCtftimeNoExists`, `badZeroAuth`.

## Rate limits

| Route | Scope | Bucket |
| --- | --- | --- |
| `PATCH /api/v1/users/me` | User | Profile name update — burst `3`, refill window `180000` ms. |
| `PATCH /api/v2/users/me` | User | Profile name update — burst `3`, refill window `180000` ms. |
| `PATCH /api/v2/users/me/avatar` | User | Avatar upload — burst `2`, refill window `120000` ms. |

The profile name update bucket is shared between V1 and V2 PATCH routes per user. See the global [rate limit summary](/api/#rate-limits) and the [response reference](/api/responses) for envelope and error shapes.
