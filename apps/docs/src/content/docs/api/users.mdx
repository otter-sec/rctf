title: Users
description: API reference for public profiles, self profiles, profile updates, avatars, email auth, CTFtime auth, and team members.

User routes expose team profiles and account-level mutations. V2 adds avatar, country, and status fields. V1 keeps team members and CTFtime account mutation routes for compatibility.

## V2 routes

| Method | Path | Auth | Gate | Request |
| --- | --- | --- | --- | --- |
| `GET{:http}` | `/api/v2/users/:id` | Public, optional admin bypass | Started | Path params |
| `GET{:http}` | `/api/v2/users/me` | Required | None | None |
| `PATCH{:http}` | `/api/v2/users/me` | Required | None | JSON body |
| `PATCH{:http}` | `/api/v2/users/me/avatar` | Required | None | Form data |
| `PUT{:http}` | `/api/v2/users/me/auth/email` | Required | None | JSON body |

`GET /api/v2/users/:id` can be read before the CTF starts by an admin token with `challsRead`.

## Public profile

`GET /api/v2/users/:id` returns public team data:

| Field | Type | Notes |
| --- | --- | --- |
| `name` | `string` | Team display name. |
| `ctftimeId` | `string \| null \| undefined` | Linked CTFtime ID when present. |
| `division` | `string` | Team division. |
| `score` | `number` | Cached current score. |
| `globalPlace` | `number \| null` | Cached global rank. |
| `divisionPlace` | `number \| null` | Cached division rank. |
| `solves` | `UserSolveV2[]` | Solved challenges visible to the route. |
| `avatarUrl` | `string \| null` | Team avatar URL. |
| `countryCode` | `string \| null` | ISO 3166-like region code from `@rctf/util`. |
| `statusText` | `string \| null` | Team status text. |

`UserSolveV2` has these fields:

| Field | Type |
| --- | --- |
| `category` | `string` |
| `name` | `string` |
| `points` | `number \| null` |
| `solves` | `number \| null` |
| `id` | `string` |
| `createdAt` | `number` |
| `bloodIndex` | `number \| null` |

## Self profile

`GET /api/v2/users/me` returns the public profile fields plus account-private data:

| Field | Type | Notes |
| --- | --- | --- |
| `id` | `string` | Team ID. |
| `email` | `string \| null` | Email address when set. |
| `teamToken` | `string` | Long-lived team token. Treat it as a credential. |
| `allowedDivisions` | `string[]` | Divisions available to the team based on ACLs. |
| `perms` | `number \| null` | Admin permission bitmask. |

## `PATCH /api/v2/users/me`

Updates profile fields:

```json
{
  "name": "new team name",
  "division": "open",
  "countryCode": "US",
  "statusText": "solving web"
}
```

| Field | Required | Notes |
| --- | --- | --- |
| `name` | No | 2 to 64 printable ASCII characters. Name updates are rate-limited. |
| `division` | No | Must be allowed for the team's current email under division ACLs. |
| `countryCode` | No | Region code or `null`. |
| `statusText` | No | String up to 60 characters or `null`. |

`goodUserUpdate` returns:

```json
{
  "user": {
    "name": "new team name",
    "email": "team@example.com",
    "division": "open",
    "avatarUrl": null,
    "countryCode": "US",
    "statusText": "solving web"
  }
}
```

The route recalculates cached user data and triggers a leaderboard update. Name updates share the profile update rate limit.

## `PATCH /api/v2/users/me/avatar`

Uploads or removes the team avatar. The route expects `multipart/form-data`:

| Form field | Required | Notes |
| --- | --- | --- |
| `avatar` | No | Image file. When omitted and an avatar exists, the current avatar is deleted. |
| `captchaCode` | No | Checked only when captcha protects `avatarUpload`. |

The API checks `maxAvatarSize`, resizes the image to 256 by 256 pixels, converts it to WebP, optionally sends it to the moderation provider, uploads it through the active upload provider, and deletes the previous avatar when replacing it.

`goodAvatarUpdated` returns `{ url: string \| null }`.

## `PUT /api/v2/users/me/auth/email`

Updates the account email:

```json
{
  "email": "team@example.com",
  "captchaCode": "optional-captcha-code"
}
```

When email delivery is configured, the route sends a verification email and returns `goodVerifySent`. When email delivery is not configured, it updates the email immediately and returns `goodEmailSet`.

The route rejects emails that would move the team outside the division ACL and rejects duplicate emails.

### V1 profile differences

V1 public and self profile responses omit `avatarUrl`, `countryCode`, `statusText`, and `bloodIndex`. V1 update only accepts `name` and `division`.

### Team members

Team members are informational email rows under the team account. They do not create separate login identities.

`POST /api/v1/users/me/members` accepts:

```json
{
  "email": "member@example.com"
}
```

The route returns `badEndpoint` when `userMembers` is disabled. Duplicate member emails return `badKnownEmail`. Teams are limited by `maxMembers`.

### CTFtime auth mutation

`PUT /api/v1/users/me/auth/ctftime` accepts:

```json
{
  "ctftimeToken": "<ctftime-token>"
}
```

The route returns `badEndpoint` when CTFtime auth is not configured. Removing the last remaining auth method returns `badZeroAuth`, which preserves the invariant that every team has at least one of email or CTFtime auth.

## Rate limits

| Route | Scope | Bucket |
| --- | --- | --- |
| `PATCH /api/v1/users/me` | User | Name updates use burst `3`, refill window `180000` ms. |
| `PATCH /api/v2/users/me` | User | Name updates use burst `3`, refill window `180000` ms. |
| `PATCH /api/v2/users/me/avatar` | User | Avatar uploads use burst `2`, refill window `120000` ms. |
