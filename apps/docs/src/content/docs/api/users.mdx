title: Users
description: API reference for public profiles, self profiles, profile updates, avatars, email auth, CTFtime auth, and team members.
---

User routes expose team profiles and account-level mutations. V2 adds avatar, country, and status fields. V1 keeps team members and CTFtime account mutation routes for compatibility.

## Routes

| Method | Path | Version | Auth | Gate |
| --- | --- | --- | --- | --- |
| `<route>GET</route>` | `<route>/api/v2/users/:id</route>` | V2 | Public, optional admin bypass | Started |
| `<route>GET</route>` | `<route>/api/v2/users/me</route>` | V2 | Required | None |
| `<route>PATCH</route>` | `<route>/api/v2/users/me</route>` | V2 | Required | None |
| `<route>PATCH</route>` | `<route>/api/v2/users/me/avatar</route>` | V2 | Required | None |
| `<route>PUT</route>` | `<route>/api/v2/users/me/auth/email</route>` | V2 | Required | None |
| `<route>GET</route>` | `<route>/api/v1/users/:id</route>` | V1 | Public, optional admin bypass | Started |
| `<route>GET</route>` | `<route>/api/v1/users/me</route>` | V1 | Required | None |
| `<route>PATCH</route>` | `<route>/api/v1/users/me</route>` | V1 | Required | None |
| `<route>GET</route>` | `<route>/api/v1/users/me/members</route>` | V1 | Required | None |
| `<route>POST</route>` | `<route>/api/v1/users/me/members</route>` | V1 | Required | None |

## V2 routes

### `<route>GET /api/v2/users/:id</route>`

Returns the public profile of a team.

| Property       | Value                                      |
| -------------- | ------------------------------------------ |
| Auth           | Public, optional admin bypass              |
| Gate           | Started                                    |
| Permissions    | `challsRead{:ts}` (start-time bypass only) |
| Captcha action | -                                          |
| Rate limit     | -                                          |

**Path params**

| Field | Type          | Required | Notes    |
| ----- | ------------- | -------- | -------- |
| `id`  | `string{:ts}` | Yes      | Team ID. |

**Response**

`<response>200 goodUserData</response>`:

```json
{
  "kind": "goodUserData",
  "message": "The user's data was successfully retrieved.",
  "data": {
    "name": "otter-sec",
    "ctftimeId": null,
    "division": "open",
    "score": 1337,
    "globalPlace": 1,
    "divisionPlace": 1,
    "solves": [],
    "avatarUrl": null,
    "countryCode": "US",
    "statusText": "solving web"
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `name` | `string{:ts}` | Team display name. |
| `ctftimeId` | `string \| null \| undefined{:ts}` | Linked CTFtime ID when present. |
| `division` | `string{:ts}` | Team division. |
| `score` | `number{:ts}` | Cached current score. |
| `globalPlace` | `number \| null{:ts}` | Cached global rank. |
| `divisionPlace` | `number \| null{:ts}` | Cached division rank. |
| `solves` | `UserSolveV2[]{:ts}` | Solved challenges visible to the route. See [`UserSolveV2`](#usersolvev2). |
| `avatarUrl` | `string \| null{:ts}` | Team avatar URL. |
| `countryCode` | `string \| null{:ts}` | ISO 3166-like region code from `@rctf/util`. |
| `statusText` | `string \| null{:ts}` | Team status text. |

**Errors**: `<response>404 badUnknownUser</response>`, `<response>401 badNotStarted</response>`.

### `<route>GET /api/v2/users/me</route>`

Returns the authenticated team's own profile, including account-private fields.

| Property       | Value    |
| -------------- | -------- |
| Auth           | Required |
| Gate           | None     |
| Permissions    | -        |
| Captcha action | -        |
| Rate limit     | -        |

**Response**

`<response>200 goodUserSelfData</response>` returns the public profile fields plus account-private data:

```json
{
  "kind": "goodUserSelfData",
  "message": "The user's own data was successfully retrieved.",
  "data": {
    "name": "otter-sec",
    "ctftimeId": null,
    "division": "open",
    "score": 1337,
    "globalPlace": 1,
    "divisionPlace": 1,
    "solves": [],
    "avatarUrl": null,
    "countryCode": "US",
    "statusText": "solving web",
    "id": "<dim><team-id></dim>",
    "email": "team@example.com",
    "teamToken": "<dim><team-token></dim>",
    "allowedDivisions": ["open"],
    "perms": null
  }
}
```

| Field              | Type                  | Notes                                            |
| ------------------ | --------------------- | ------------------------------------------------ |
| `id`               | `string{:ts}`         | Team ID.                                         |
| `email`            | `string \| null{:ts}` | Email address when set.                          |
| `teamToken`        | `string{:ts}`         | Long-lived team token. Treat it as a credential. |
| `allowedDivisions` | `string[]{:ts}`       | Divisions available to the team based on ACLs.   |
| `perms`            | `number \| null{:ts}` | Admin permission bitmask.                        |

All other fields match the [public profile](#get-apiv2usersid).

**Errors**: `<response>401 badToken</response>`.

### `<route>PATCH /api/v2/users/me</route>`

Updates profile fields. Recalculates cached user data and triggers a leaderboard update.

| Property       | Value                                                                |
| -------------- | -------------------------------------------------------------------- |
| Auth           | Required                                                             |
| Gate           | None                                                                 |
| Permissions    | -                                                                    |
| Captcha action | -                                                                    |
| Rate limit     | Profile name update - burst `3`, refill window `180000` ms per user. |

**Request body**

```json
{
  "name": "new team name",
  "division": "open",
  "countryCode": "US",
  "statusText": "solving web"
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `name` | `string{:ts}` | No | 2–64 printable ASCII characters. Name updates are rate-limited. |
| `division` | `string{:ts}` | No | Must be allowed for the team's current email under division ACLs. |
| `countryCode` | `string \| null{:ts}` | No | Region code or `null{:ts}`. |
| `statusText` | `string \| null{:ts}` | No | String up to 60 characters or `null{:ts}`. |

**Response**

`<response>200 goodUserUpdate</response>`:

```json
{
  "kind": "goodUserUpdate",
  "message": "The user was successfully updated.",
  "data": {
    "user": {
      "name": "new team name",
      "email": "team@example.com",
      "division": "open",
      "avatarUrl": null,
      "countryCode": "US",
      "statusText": "solving web"
    }
  }
}
```

| Field              | Type                  | Notes                 |
| ------------------ | --------------------- | --------------------- |
| `user.name`        | `string{:ts}`         | Updated team name.    |
| `user.email`       | `string \| null{:ts}` | Current email.        |
| `user.division`    | `string{:ts}`         | Current division.     |
| `user.avatarUrl`   | `string \| null{:ts}` | Current avatar URL.   |
| `user.countryCode` | `string \| null{:ts}` | Current country code. |
| `user.statusText`  | `string \| null{:ts}` | Current status text.  |

**Errors**: `<response>400 badBody</response>`, `<response>400 badName</response>`, `<response>409 badKnownName</response>`, `<response>403 badDivisionNotAllowed</response>`, `<response>429 badRateLimit</response>`.

### `<route>PATCH /api/v2/users/me/avatar</route>`

Uploads or removes the team avatar. The route expects `multipart/form-data`.

| Property       | Value                                                          |
| -------------- | -------------------------------------------------------------- |
| Auth           | Required                                                       |
| Gate           | None                                                           |
| Permissions    | -                                                              |
| Captcha action | `avatarUpload{:ts}` (when configured)                          |
| Rate limit     | Avatar upload - burst `2`, refill window `120000` ms per user. |

**Request**

| Form field | Type | Required | Notes |
| --- | --- | --- | --- |
| `avatar` | file | No | Image file. When omitted and an avatar exists, the current avatar is deleted. |
| `captchaCode` | `string{:ts}` | No | Checked only when captcha protects `avatarUpload{:ts}`. |

The route checks `maxAvatarSize`, resizes the image to 256×256 pixels, converts it to WebP, optionally sends it to the moderation provider, uploads it through the active upload provider, and deletes the previous avatar when replacing it.

**Response**

`<response>200 goodAvatarUpdated</response>`:

```json
{
  "kind": "goodAvatarUpdated",
  "message": "The avatar was successfully updated.",
  "data": {
    "url": "https://uploads.example.com/avatars/<id>.webp"
  }
}
```

| Field | Type                  | Notes                                                       |
| ----- | --------------------- | ----------------------------------------------------------- |
| `url` | `string \| null{:ts}` | New avatar URL, or `null{:ts}` when the avatar was deleted. |

**Errors**: `<response>400 badBody</response>`, `<response>400 badAvatarFile</response>`, `<response>400 badAvatarFileSize</response>`, `<response>400 badModerationNotPassed</response>`, `<response>500 badFilesUpload</response>`, `<response>403 badCaptcha</response>`, `<response>429 badRateLimit</response>`.

### `<route>PUT /api/v2/users/me/auth/email</route>`

Updates the account email. When email delivery is configured, the route sends a verification email and returns `<response>200 goodVerifySent</response>`. When email delivery is not configured, it updates the email immediately and returns `<response>200 goodEmailSet</response>`.

| Property       | Value                             |
| -------------- | --------------------------------- |
| Auth           | Required                          |
| Gate           | None                              |
| Permissions    | -                                 |
| Captcha action | `setEmail{:ts}` (when configured) |
| Rate limit     | -                                 |

**Request body**

```json
{
  "email": "team@example.com",
  "captchaCode": "optional-captcha-code"
}
```

| Field         | Type          | Required | Notes                                               |
| ------------- | ------------- | -------- | --------------------------------------------------- |
| `email`       | `string{:ts}` | Yes      | New email address.                                  |
| `captchaCode` | `string{:ts}` | No       | Checked only when captcha protects `setEmail{:ts}`. |

The route rejects emails that would move the team outside the division ACL (`<response>403 badEmailChangeDivision</response>`) and rejects duplicate emails (`<response>409 badKnownEmail</response>`).

**Response**

Behavior branches on email provider configuration:

| Condition | Success kind | Effect |
| --- | --- | --- |
| Email delivery configured | `<response>200 goodVerifySent</response>` | Sends a verification email. Consume the token via [`<route>POST /api/v2/auth/verify</route>`](/api/auth/verify/). |
| Email delivery not configured | `<response>200 goodEmailSet</response>` | Updates the email immediately. |

Both kinds return no data.

**Errors**: `<response>400 badBody</response>`, `<response>400 badEmail</response>`, `<response>409 badKnownEmail</response>`, `<response>403 badEmailChangeDivision</response>`, `<response>403 badCaptcha</response>`, `<response>409 badZeroAuth</response>`.

## V1 compatibility

V1 routes remain available for legacy clients. Behavior matches V2 except where noted below. Shared response and error semantics apply unless overridden.

### `<route>GET /api/v1/users/:id</route>`

Differs from V2: response omits `avatarUrl`, `countryCode`, `statusText`, and `bloodIndex` (on each entry in `solves`).

### `<route>GET /api/v1/users/me</route>`

Differs from V2: response omits `avatarUrl`, `countryCode`, and `statusText`.

### `<route>PATCH /api/v1/users/me</route>`

Differs from V2: only accepts `name` and `division`. Shares the profile name update rate limit (burst `3`, refill window `180000` ms per user) with [`<route>PATCH /api/v2/users/me</route>`](#patch-apiv2usersme).

### `<route>PUT /api/v1/users/me/auth/email</route>`

Differs from V2: V1 request body uses the V1 captcha field name (`recaptchaCode`). Response semantics match V2 - `<response>200 goodVerifySent</response>` when email delivery is configured, `<response>200 goodEmailSet</response>` otherwise. Same division-ACL and duplicate-email rejections apply.

### `<route>DELETE /api/v1/users/me/auth/email</route>`

V1-only. Removes the account email. Preserves the `<response>409 badZeroAuth</response>` invariant: a team must keep at least one of email or CTFtime auth.

| Property       | Value    |
| -------------- | -------- |
| Auth           | Required |
| Gate           | None     |
| Permissions    | -        |
| Captcha action | -        |
| Rate limit     | -        |

**Response**

`<response>200 goodEmailRemoved</response>` (no data).

**Errors**: `<response>404 badEmailNoExists</response>`, `<response>409 badZeroAuth</response>`.

### `<route>PUT /api/v1/users/me/auth/ctftime</route>`

V1-only. Links the account to a CTFtime team via a CTFtime auth token.

| Property       | Value    |
| -------------- | -------- |
| Auth           | Required |
| Gate           | None     |
| Permissions    | -        |
| Captcha action | -        |
| Rate limit     | -        |

**Request body**

```json
{
  "ctftimeToken": "<dim><ctftime-token></dim>"
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `ctftimeToken` | `string{:ts}` | Yes | CTFtime auth token. Returns `<response>404 badEndpoint</response>` when CTFtime auth is not configured. |

**Response**

`<response>200 goodCtftimeAuthSet</response>` (no data).

**Errors**: `<response>400 badBody</response>`, `<response>404 badEndpoint</response>`, `<response>401 badCtftimeToken</response>`, `<response>409 badKnownCtftimeId</response>`.

### `<route>DELETE /api/v1/users/me/auth/ctftime</route>`

V1-only. Unlinks the account's CTFtime auth. Removing the last remaining auth method returns `<response>409 badZeroAuth</response>`, which preserves the invariant that every team has at least one of email or CTFtime auth.

| Property       | Value    |
| -------------- | -------- |
| Auth           | Required |
| Gate           | None     |
| Permissions    | -        |
| Captcha action | -        |
| Rate limit     | -        |

**Response**

`<response>200 goodCtftimeRemoved</response>` (no data).

**Errors**: `<response>404 badCtftimeNoExists</response>`, `<response>409 badZeroAuth</response>`.

### `<route>GET /api/v1/users/me/members</route>`

V1-only. Lists informational team-member email rows under the team account. Team members do not create separate login identities. Returns `<response>404 badEndpoint</response>` when `userMembers` is disabled.

| Property       | Value    |
| -------------- | -------- |
| Auth           | Required |
| Gate           | None     |
| Permissions    | -        |
| Captcha action | -        |
| Rate limit     | -        |

**Response**

`<response>200 goodMemberData</response>` returns a team member array.

**Errors**: `<response>404 badEndpoint</response>`.

### `<route>POST /api/v1/users/me/members</route>`

V1-only. Adds a team member email row. Teams are limited by `maxMembers`.

| Property       | Value    |
| -------------- | -------- |
| Auth           | Required |
| Gate           | None     |
| Permissions    | -        |
| Captcha action | -        |
| Rate limit     | -        |

**Request body**

```json
{
  "email": "member@example.com"
}
```

| Field   | Type          | Required | Notes                 |
| ------- | ------------- | -------- | --------------------- |
| `email` | `string{:ts}` | Yes      | Member email address. |

**Response**

`<response>200 goodMemberCreate</response>` returns the created team member object.

**Errors**: `<response>400 badBody</response>`, `<response>400 badEmail</response>`, `<response>409 badKnownEmail</response>`, `<response>409 badTooManyMembers</response>`, `<response>404 badEndpoint</response>`.

### `<route>DELETE /api/v1/users/me/members/:id</route>`

## Rate limits

| Route | Scope | Bucket |
| --- | --- | --- |
| `<route>PATCH /api/v1/users/me</route>` | User | Profile name update - burst `3`, refill window `180000` ms. |
| `<route>PATCH /api/v2/users/me</route>` | User | Profile name update - burst `3`, refill window `180000` ms. |
| `<route>PATCH /api/v2/users/me/avatar</route>` | User | Avatar upload - burst `2`, refill window `120000` ms. |

The profile name update bucket is shared between V1 and V2 PATCH routes per user. See the global [rate limit summary](/api/#rate-limits) and the [response reference](/api/responses/) for envelope and error shapes.
