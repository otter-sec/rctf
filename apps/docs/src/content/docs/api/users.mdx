User routes expose team profiles and account-level mutations. V2 adds avatar, country, and status fields. V1 keeps team members and CTFtime account mutation routes for compatibility.

## V2 routes

### `[36mGET[0m [2m/api/[22mv2[2m/[22musers[2m/[22m:id{:ansi}`

Returns the public profile of a team.

| Property       | Value                                      |
| -------------- | ------------------------------------------ |
| Auth           | Public, optional admin bypass              |
| Gate           | Started                                    |
| Permissions    | `challsRead{:ts}` (start-time bypass only) |
| Captcha action | —                                          |
| Rate limit     | —                                          |

**Path params**

| Field | Type          | Required | Notes    |
| ----- | ------------- | -------- | -------- |
| `id`  | `string{:ts}` | Yes      | Team ID. |

**Response**

`[2;32m200 [22mgoodUserData[0m{:ansi}`:

```json
{
  "kind": "goodUserData",
  "message": "The user's data was successfully retrieved.",
  "data": {
    "name": "maple bacon",
    "ctftimeId": null,
    "division": "open",
    "score": 1337,
    "globalPlace": 1,
    "divisionPlace": 1,
    "solves": [],
    "avatarUrl": null,
    "countryCode": "US",
    "statusText": "solving web"
  }
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `name` | `string{:ts}` | Team display name. |
| `ctftimeId` | `string \| null \| undefined{:ts}` | Linked CTFtime ID when present. |
| `division` | `string{:ts}` | Team division. |
| `score` | `number{:ts}` | Cached current score. |
| `globalPlace` | `number \| null{:ts}` | Cached global rank. |
| `divisionPlace` | `number \| null{:ts}` | Cached division rank. |
| `solves` | `UserSolveV2[]{:ts}` | Solved challenges visible to the route. See [`UserSolveV2`](#usersolvev2). |
| `avatarUrl` | `string \| null{:ts}` | Team avatar URL. |
| `countryCode` | `string \| null{:ts}` | ISO 3166-like region code from `@rctf/util`. |
| `statusText` | `string \| null{:ts}` | Team status text. |

**Errors**: `[2;31m404 [22mbadUnknownUser[0m{:ansi}`, `[2;31m401 [22mbadNotStarted[0m{:ansi}`.

### `[33mPATCH[0m [2m/api/[22mv2[2m/[22musers[2m/[22mme{:ansi}`

Updates profile fields. Recalculates cached user data and triggers a leaderboard update.

| Property       | Value                                                                |
| -------------- | -------------------------------------------------------------------- |
| Auth           | Required                                                             |
| Gate           | None                                                                 |
| Permissions    | —                                                                    |
| Captcha action | —                                                                    |
| Rate limit     | Profile name update — burst `3`, refill window `180000` ms per user. |

**Request body**

```json
{
  "name": "new team name",
  "division": "open",
  "countryCode": "US",
  "statusText": "solving web"
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `name` | `string{:ts}` | No | 2–64 printable ASCII characters. Name updates are rate-limited. |
| `division` | `string{:ts}` | No | Must be allowed for the team's current email under division ACLs. |
| `countryCode` | `string \| null{:ts}` | No | Region code or `null{:ts}`. |
| `statusText` | `string \| null{:ts}` | No | String up to 60 characters or `null{:ts}`. |

**Response**

`[2;32m200 [22mgoodUserUpdate[0m{:ansi}`:

```json
{
  "kind": "goodUserUpdate",
  "message": "The user was successfully updated.",
  "data": {
    "user": {
      "name": "new team name",
      "email": "team@example.com",
      "division": "open",
      "avatarUrl": null,
      "countryCode": "US",
      "statusText": "solving web"
    }
  }
}
```

| Field              | Type                  | Notes                 |
| ------------------ | --------------------- | --------------------- |
| `user.name`        | `string{:ts}`         | Updated team name.    |
| `user.email`       | `string \| null{:ts}` | Current email.        |
| `user.division`    | `string{:ts}`         | Current division.     |
| `user.avatarUrl`   | `string \| null{:ts}` | Current avatar URL.   |
| `user.countryCode` | `string \| null{:ts}` | Current country code. |
| `user.statusText`  | `string \| null{:ts}` | Current status text.  |

**Errors**: `[2;31m400 [22mbadBody[0m{:ansi}`, `[2;31m400 [22mbadName[0m{:ansi}`, `[2;31m409 [22mbadKnownName[0m{:ansi}`, `[2;31m403 [22mbadDivisionNotAllowed[0m{:ansi}`, `[2;31m429 [22mbadRateLimit[0m{:ansi}`.

### `[33mPATCH[0m [2m/api/[22mv2[2m/[22musers[2m/[22mme[2m/[22mavatar{:ansi}`

Uploads or removes the team avatar. The route expects `multipart/form-data`.

| Property       | Value                                                          |
| -------------- | -------------------------------------------------------------- |
| Auth           | Required                                                       |
| Gate           | None                                                           |
| Permissions    | —                                                              |
| Captcha action | `avatarUpload{:ts}` (when configured)                          |
| Rate limit     | Avatar upload — burst `2`, refill window `120000` ms per user. |

**Request**

| Form field | Type | Required | Notes |
| --- | --- | --- | --- |
| `avatar` | file | No | Image file. When omitted and an avatar exists, the current avatar is deleted. |
| `captchaCode` | `string{:ts}` | No | Checked only when captcha protects `avatarUpload{:ts}`. |

The route checks `maxAvatarSize`, resizes the image to 256×256 pixels, converts it to WebP, optionally sends it to the moderation provider, uploads it through the active upload provider, and deletes the previous avatar when replacing it.

**Response**

`[2;32m200 [22mgoodAvatarUpdated[0m{:ansi}`:

```json
{
  "kind": "goodAvatarUpdated",
  "message": "The avatar was successfully updated.",
  "data": {
    "url": "https://uploads.example.com/avatars/<id>.webp"
  }
}
```

| Field | Type                  | Notes                                                       |
| ----- | --------------------- | ----------------------------------------------------------- |
| `url` | `string \| null{:ts}` | New avatar URL, or `null{:ts}` when the avatar was deleted. |

**Errors**: `[2;31m400 [22mbadBody[0m{:ansi}`, `[2;31m400 [22mbadAvatarFile[0m{:ansi}`, `[2;31m400 [22mbadAvatarFileSize[0m{:ansi}`, `[2;31m400 [22mbadModerationNotPassed[0m{:ansi}`, `[2;31m500 [22mbadFilesUpload[0m{:ansi}`, `[2;31m403 [22mbadCaptcha[0m{:ansi}`, `[2;31m429 [22mbadRateLimit[0m{:ansi}`.

### `[35mPUT[0m [2m/api/[22mv2[2m/[22musers[2m/[22mme[2m/[22mauth[2m/[22memail{:ansi}`

Updates the account email. When email delivery is configured, the route sends a verification email and returns `[2;32m200 [22mgoodVerifySent[0m{:ansi}`. When email delivery is not configured, it updates the email immediately and returns `[2;32m200 [22mgoodEmailSet[0m{:ansi}`.

| Property       | Value                             |
| -------------- | --------------------------------- |
| Auth           | Required                          |
| Gate           | None                              |
| Permissions    | —                                 |
| Captcha action | `setEmail{:ts}` (when configured) |
| Rate limit     | —                                 |

**Request body**

```json
{
  "email": "team@example.com",
  "captchaCode": "optional-captcha-code"
}
```

| Field         | Type          | Required | Notes                                               |
| ------------- | ------------- | -------- | --------------------------------------------------- |
| `email`       | `string{:ts}` | Yes      | New email address.                                  |
| `captchaCode` | `string{:ts}` | No       | Checked only when captcha protects `setEmail{:ts}`. |

The route rejects emails that would move the team outside the division ACL (`[2;31m403 [22mbadEmailChangeDivision[0m{:ansi}`) and rejects duplicate emails (`[2;31m409 [22mbadKnownEmail[0m{:ansi}`).

**Response**

Behavior branches on email provider configuration:

| Condition | Success kind | Effect |
| --- | --- | --- |
| Email delivery configured | `[2;32m200 [22mgoodVerifySent[0m{:ansi}` | Sends a verification email; consume the token via [`[32mPOST[0m [2m/api/[22mv2[2m/[22mauth[2m/[22mverify{:ansi}`](/api/auth/#post-apiv2authverify). |
| Email delivery not configured | `[2;32m200 [22mgoodEmailSet[0m{:ansi}` | Updates the email immediately. |

Both kinds return no data.

**Errors**: `[2;31m400 [22mbadBody[0m{:ansi}`, `[2;31m400 [22mbadEmail[0m{:ansi}`, `[2;31m409 [22mbadKnownEmail[0m{:ansi}`, `[2;31m403 [22mbadEmailChangeDivision[0m{:ansi}`, `[2;31m403 [22mbadCaptcha[0m{:ansi}`, `[2;31m409 [22mbadZeroAuth[0m{:ansi}`.

## V1 compatibility

V1 routes remain available for legacy clients. Behavior matches V2 except where noted below. Shared response and error semantics apply unless overridden.

### `[36mGET[0m [2m/api/[22mv1[2m/[22musers[2m/[22m:id{:ansi}`

Differs from V2: response omits `avatarUrl`, `countryCode`, `statusText`, and `bloodIndex` (on each entry in `solves`).

### `[36mGET[0m [2m/api/[22mv1[2m/[22musers[2m/[22mme{:ansi}`

Differs from V2: response omits `avatarUrl`, `countryCode`, and `statusText`.

### `[33mPATCH[0m [2m/api/[22mv1[2m/[22musers[2m/[22mme{:ansi}`

Differs from V2: only accepts `name` and `division`. Shares the profile name update rate limit (burst `3`, refill window `180000` ms per user) with [`[33mPATCH[0m [2m/api/[22mv2[2m/[22musers[2m/[22mme{:ansi}`](#patch-apiv2usersme).

### `[35mPUT[0m [2m/api/[22mv1[2m/[22musers[2m/[22mme[2m/[22mauth[2m/[22memail{:ansi}`

Differs from V2: V1 request body uses the V1 captcha field name (`recaptchaCode`). Response semantics match V2 — `[2;32m200 [22mgoodVerifySent[0m{:ansi}` when email delivery is configured, `[2;32m200 [22mgoodEmailSet[0m{:ansi}` otherwise. Same division-ACL and duplicate-email rejections apply.
