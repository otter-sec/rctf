Authentication is team-based. A team owns one account, one team token, and any number of auth tokens. Every route on this page is public except `[36mGET[0m [2m/api/[22mv1[2m/[22mauth[2m/[22mtest{:ansi}`, which validates an auth token.

## V2 routes

### `[32mPOST[0m [2m/api/[22mv2[2m/[22mauth[2m/[22mregister{:ansi}`

Registers a team. Returns auth tokens immediately when verification is not required, otherwise sends a verification email.

| Property       | Value                             |
| -------------- | --------------------------------- |
| Auth           | Public                            |
| Gate           | None                              |
| Permissions    | —                                 |
| Captcha action | `register{:ts}` (when configured) |
| Rate limit     | —                                 |

**Request body**

```json
{
  "name": "maple bacon",
  "email": "team@example.com",
  "ctftimeToken": "optional-ctftime-token",
  "captchaCode": "optional-captcha-code"
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `name` | `string{:ts}` | Yes | 2–64 printable ASCII characters. |
| `email` | `string{:ts}` | Conditional | Required when `ctftimeToken` is omitted. |
| `ctftimeToken` | `string{:ts}` | Conditional | Required when `email` is omitted. Returns `[2;31m404 [22mbadEndpoint[0m{:ansi}` when CTFtime is not configured. |
| `captchaCode` | `string{:ts}` | No | Checked only when captcha protects `register{:ts}`. |

The route assigns the team to the default allowed division based on email ACLs. CTFtime registrations bypass ACLs entirely.

**Response**

When registration completes immediately (CTFtime auth, or no email provider is configured), returns `[2;32m200 [22mgoodRegisterV2[0m{:ansi}`:

```json
{
  "kind": "goodRegisterV2",
  "message": "The user was created.",
  "data": {
    "authToken": "<auth-token>",
    "teamToken": "<team-token>"
  }
}
```

When email verification is required, returns `[2;32m200 [22mgoodVerifySent[0m{:ansi}` (no data) and sends a verification email.

**Errors**: `[2;31m400 [22mbadBody[0m{:ansi}`, `[2;31m400 [22mbadEmail[0m{:ansi}`, `[2;31m400 [22mbadName[0m{:ansi}`, `[2;31m409 [22mbadKnownEmail[0m{:ansi}`, `[2;31m409 [22mbadKnownName[0m{:ansi}`, `[2;31m409 [22mbadKnownCtftimeId[0m{:ansi}`, `[2;31m403 [22mbadCompetitionNotAllowed[0m{:ansi}`, `[2;31m403 [22mbadDivisionNotAllowed[0m{:ansi}`, `[2;31m400 [22mbadRegistrationsDisabled[0m{:ansi}`, `[2;31m403 [22mbadCaptcha[0m{:ansi}`, `[2;31m404 [22mbadEndpoint[0m{:ansi}`, `[2;31m401 [22mbadCtftimeToken[0m{:ansi}`.

### `[32mPOST[0m [2m/api/[22mv2[2m/[22mauth[2m/[22mverify{:ansi}`

Consumes a verification token to complete a pending registration, log in with a team token, or finalize an email change.

| Property       | Value  |
| -------------- | ------ |
| Auth           | Public |
| Gate           | None   |
| Permissions    | —      |
| Captcha action | —      |
| Rate limit     | —      |

**Request body**

```json
{
  "verifyToken": "<token>"
}
```

| Field         | Type          | Required | Notes                                        |
| ------------- | ------------- | -------- | -------------------------------------------- |
| `verifyToken` | `string{:ts}` | Yes      | Pending-registration, team, or verify token. |

**Response**

Behavior branches on the token kind:

| Token kind | Success kind | Effect |
| --- | --- | --- |
| Pending registration | `[2;32m200 [22mgoodRegisterV2[0m{:ansi}` | Creates the team and returns `authToken` + `teamToken`. Claims the verification row. |
| Team | `[2;32m200 [22mgoodVerify[0m{:ansi}` | Returns a fresh `authToken`. |
| Verify | `[2;32m200 [22mgoodEmailSet[0m{:ansi}` | Updates the team email after re-checking division ACLs. |

`[2;32m200 [22mgoodRegisterV2[0m{:ansi}` data matches the [register response](#post-apiv2authregister). `[2;32m200 [22mgoodVerify[0m{:ansi}` returns `{ authToken: string }`. `[2;32m200 [22mgoodEmailSet[0m{:ansi}` returns no data.

**Errors**: `[2;31m400 [22mbadBody[0m{:ansi}`, `[2;31m401 [22mbadTokenVerification[0m{:ansi}`, `[2;31m403 [22mbadEmailChangeDivision[0m{:ansi}`, `[2;31m409 [22mbadKnownEmail[0m{:ansi}`.

### `[32mPOST[0m [2m/api/[22mv2[2m/[22mauth[2m/[22mrecover{:ansi}`

Sends an account-recovery email containing a team token. Always returns `[2;32m200 [22mgoodVerifySent[0m{:ansi}`, even when the email is unknown, to prevent account enumeration.

| Property       | Value                            |
| -------------- | -------------------------------- |
| Auth           | Public                           |
| Gate           | None                             |
| Permissions    | —                                |
| Captcha action | `recover{:ts}` (when configured) |
| Rate limit     | —                                |

**Request body**

```json
{
  "email": "team@example.com",
  "captchaCode": "optional-captcha-code"
}
```

| Field         | Type          | Required | Notes                                              |
| ------------- | ------------- | -------- | -------------------------------------------------- |
| `email`       | `string{:ts}` | Yes      | Recovery destination.                              |
| `captchaCode` | `string{:ts}` | No       | Checked only when captcha protects `recover{:ts}`. |

**Response**

`[2;32m200 [22mgoodVerifySent[0m{:ansi}` (no data).

**Errors**: `[2;31m400 [22mbadBody[0m{:ansi}`, `[2;31m400 [22mbadEmail[0m{:ansi}`, `[2;31m404 [22mbadEndpoint[0m{:ansi}` (no email provider configured), `[2;31m403 [22mbadCaptcha[0m{:ansi}`.

| Property       | Value  |
| -------------- | ------ |
| Auth           | Public |
| Gate           | None   |
| Permissions    | —      |

| Rate limit     | —      |

**Query**

| Field   | Type          | Required | Notes                                        |
| ------- | ------------- | -------- | -------------------------------------------- |

**Response**

`[2;32m200 [22mgoodVerifyInfo[0m{:ansi}`:

## V1 compatibility

V1 routes remain available for legacy clients. Behavior matches V2 except where noted below. Shared response and error semantics apply unless overridden.

### `[32mPOST[0m [2m/api/[22mv1[2m/[22mauth[2m/[22mregister{:ansi}`

Differs from V2:

- Captcha field is `recaptchaCode` instead of `captchaCode`.
- Immediate success returns `[2;32m200 [22mgoodRegister[0m{:ansi}` with `{ authToken: string }` only — no `teamToken`.

### `[32mPOST[0m [2m/api/[22mv1[2m/[22mauth[2m/[22mlogin{:ansi}`

V1-only. Exchanges a team token or a CTFtime auth token for an auth token. There is no V2 equivalent for CTFtime login; V2 clients use [`[32mPOST[0m [2m/api/[22mv2[2m/[22mauth[2m/[22mverify{:ansi}`](#post-apiv2authverify) with a team token, or the CTFtime callback flow.

| Property       | Value  |
| -------------- | ------ |
| Auth           | Public |
| Gate           | None   |
| Permissions    | —      |
| Captcha action | —      |
| Rate limit     | —      |

**Request body**

```json
{
  "teamToken": "<team-token>",
  "ctftimeToken": "<ctftime-token>"
}
```

At least one of `teamToken` or `ctftimeToken` must be present. A team token is parsed as `TokenKind.Team{:ts}`. A CTFtime token is parsed as `TokenKind.CtftimeAuth{:ts}` and matched against a team linked to that CTFtime ID.

**Response**

`[2;32m200 [22mgoodLogin[0m{:ansi}`: `{ authToken: string }`.

**Errors**: `[2;31m400 [22mbadBody[0m{:ansi}`, `[2;31m401 [22mbadTokenVerification[0m{:ansi}`, `[2;31m401 [22mbadCtftimeToken[0m{:ansi}`, `[2;31m404 [22mbadCtftimeNoExists[0m{:ansi}`, `[2;31m404 [22mbadEndpoint[0m{:ansi}` (CTFtime not configured).

### `[32mPOST[0m [2m/api/[22mv1[2m/[22mauth[2m/[22mverify{:ansi}`

Differs from V2: the pending-registration path returns `[2;32m200 [22mgoodRegister[0m{:ansi}` with `{ authToken: string }` only.

### `[32mPOST[0m [2m/api/[22mv1[2m/[22mauth[2m/[22mrecover{:ansi}`

Differs from V2: captcha field is `recaptchaCode` instead of `captchaCode`.

### `[36mGET[0m [2m/api/[22mv1[2m/[22mauth[2m/[22mtest{:ansi}`

Validates the supplied auth token. Returns `[2;32m200 [22mgoodToken[0m{:ansi}` (no data) when the token is valid, `[2;31m401 [22mbadToken[0m{:ansi}` otherwise. Useful for client-side session checks.

| Property       | Value    |
| -------------- | -------- |
| Auth           | Required |
| Gate           | None     |
| Permissions    | —        |
| Captcha action | —        |
| Rate limit     | —        |

## Token lifecycle

All tokens are AES-256-GCM encrypted with `tokenKey`. Rotating `tokenKey` invalidates every existing token.

| Token kind    | Lifetime       | Used by                                              |
| ------------- | -------------- | ---------------------------------------------------- |
| `Auth`        | No expiry      | `Authorization: Bearer <auth-token>` on user routes. |
| `Team`        | No expiry      | Recovery, V1 login, and V2 verify with a team token. |
| `Verify`      | `loginTimeout` | Email updates and pending registrations. One-time.   |
| `CtftimeAuth` | `loginTimeout` | CTFtime registration and V1 login handoff.           |

Verify tokens additionally require a one-time Redis marker, so a valid encrypted token can still return `[2;31m401 [22mbadTokenVerification[0m{:ansi}` after the marker has been consumed or expired.
