Challenge routes expose public challenge metadata, solve history, and the legacy flag submission endpoint. Admin challenge mutation routes are documented in [Admin](/api/admin).

## Routes

| Method | Path | Version | Auth | Gate | Request |
| --- | --- | --- | --- | --- | --- |
| `GET{:http}` | `/api/v2/challs` | V2 | Public, optional admin bypass | Started | None |
| `GET{:http}` | `/api/v2/challs/:id/solves` | V2 | Optional | Started | Query string |
| `GET{:http}` | `/api/v1/challs` | V1 | Public, optional admin bypass | Started | None |
| `GET{:http}` | `/api/v1/challs/:id/solves` | V1 | Public, optional admin bypass | Started | Query string |
| `POST{:http}` | `/api/v1/challs/:id/submit` | V1 | Required | Started and not ended | JSON body |

Started-gated challenge routes may be read before the CTF starts when the request includes an auth token with `challsRead`.

## `GET /api/v2/challs`

Returns the visible challenge list. Hidden challenges are omitted for non-admin users. Challenges with a future `releaseTime` are omitted until their release timestamp.

`goodChallenges` returns an array of challenge objects:

| Field | Type | Notes |
| --- | --- | --- |
| `id` | `string` | Challenge ID. |
| `name` | `string` | Display name. |
| `description` | `string` | Markdown challenge description. |
| `category` | `string` | Category name. |
| `author` | `string` | Author display name. |
| `files` | `ChallengeFileV2[]` | Downloadable files. |
| `points` | `number` | Current score after dynamic scoring. |
| `solves` | `number` | Current solve count. |
| `sortWeight` | `number \| null` | Optional ordering weight. |
| `instancerLifetime` | `number \| null` | Instance timeout in milliseconds when the challenge has instancer config and an instancer provider is enabled. |
| `instancerExtendable` | `boolean` | `false` only when challenge config explicitly disables extension. |
| `adminBotInputs` | `Record<string, RegexRule> \| null \| undefined` | Participant input schema for admin bot challenges. |
| `hasFlag` | `boolean` | Indicates whether the challenge has a flag configured. |

`ChallengeFileV2` has these fields:

| Field | Type | Notes |
| --- | --- | --- |
| `name` | `string` | Display filename. |
| `url` | `string` | Download URL. |
| `size` | `number \| null` | File size in bytes. Old data may have `null`. |

`RegexRule` has `pattern` and optional `flags`.

## `GET /api/v1/challs`

The v1 challenge list returns the same core fields without v2 integration metadata:

| Field | Type |
| --- | --- |
| `id` | `string` |
| `name` | `string` |
| `description` | `string` |
| `category` | `string` |
| `author` | `string` |
| `files` | `{ name: string, url: string }[]` |
| `points` | `number` |
| `solves` | `number` |
| `sortWeight` | `number \| null \| undefined` |

## `GET /api/v2/challs/:id/solves`

Returns paginated solve history for one challenge:

| Query | Required | Notes |
| --- | --- | --- |
| `limit` | Yes | Integer greater than or equal to `1`. The config enforces the maximum. |
| `offset` | Yes | Integer greater than or equal to `0`. The config enforces the maximum. |

`goodChallengeSolves` returns:

```json
{
  "solves": [
    {
      "id": "solve-id",
      "createdAt": 1710000000000,
      "userId": "team-id",
      "userName": "maple bacon",
      "userAvatarUrl": null,
      "userCountryCode": "US",
      "userStatusText": null,
      "globalPlace": 3,
      "division": "open",
      "divisionPlace": 1,
      "bloodIndex": 0
    }
  ],
  "mySolvePosition": 12
}
```

| Field | Type | Notes |
| --- | --- | --- |
| `solves[]` | `ChallengeSolveV2[]` | Solve rows ordered by solve time. |
| `mySolvePosition` | `number \| null` | The authenticated user's solve position for this challenge when optional auth is present. |
| `bloodIndex` | `number \| null` | `0`, `1`, or `2` for the first three solves. Otherwise `null`. |

## `GET /api/v1/challs/:id/solves`

V1 solve history uses the same query parameters and returns less user profile data:

```json
{
  "solves": [
    {
      "id": "solve-id",
      "createdAt": 1710000000000,
      "userId": "team-id",
      "userName": "maple bacon"
    }
  ]
}
```

## `POST /api/v1/challs/:id/submit`

Submits a flag for the authenticated team:

```json
{
  "flag": "flag{example}"
}
```

| Field | Required | Notes |
| --- | --- | --- |
| `flag` | Yes | String with maximum length `1024`. |

Correct flags return `goodFlag` and record a solve. Incorrect flags return `badFlag`. Duplicate solves return `badAlreadySolvedChallenge`.

Submissions are rate-limited per user and challenge. A rate-limited request returns `badRateLimit` with `data.timeLeft` in milliseconds.

The route records a submission audit row for both correct and incorrect attempts. Admins can read those rows through `/api/v2/admin/submissions`.

## Scoring behavior

Challenge point values are computed by the configured [scoring provider](/providers/scores). The public challenge list returns the current point value, not the configured `points.min` or `points.max` range. Challenge updates and solve mutations trigger leaderboard recalculation.

## Visibility behavior

Non-admin users only see challenges that are not hidden and whose release time has passed. Admin users with `challsRead` can bypass the CTF start gate through the same public routes, but hidden and scheduled release handling is still applied by the challenge service according to the route implementation.
