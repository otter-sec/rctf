title: Responses
description: Reference for rCTF response envelopes, common errors, response payloads, and shared API object shapes.
---

Response definitions live in `@rctf/types` under `packages/types/src/responses`. Each definition fixes the response `kind`, HTTP status, message, and optional `data` schema.

## Envelope

Responses with data use:

```json
{
  "kind": "goodUserSelfData",
  "message": "The user's own data was successfully retrieved.",
  "data": {}
}
```

Responses without data omit `data`:

```json
{
  "kind": "goodToken",
  "message": "The authorization token is valid."
}
```

Global 404 errors return `[2;31m404 [22mbadEndpoint[0m{:ansi}`. Unhandled route errors return `[2;31m500 [22merrorInternal[0m{:ansi}`.
