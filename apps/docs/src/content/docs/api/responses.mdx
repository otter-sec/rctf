---
title: Responses
description: Reference for rCTF response envelopes, common errors, response payloads, and shared API object shapes.
---

Response definitions live in `@rctf/types` under `packages/types/src/responses`. Each definition fixes the response `kind`, HTTP status, message, and optional `data` schema.

## Envelope

Responses with data use:

```json
{
  "kind": "goodUserSelfData",
  "message": "The user's own data was successfully retrieved.",
  "data": {}
}
```

Responses without data omit `data`:

```json
{
  "kind": "goodToken",
  "message": "The authorization token is valid."
}
```

Global 404 errors return `<red>404 badEndpoint</red>`. Unhandled route errors return `<red>500 errorInternal</red>`.

## Common errors

| Kind | Status | Data | Meaning |
| --- | --- | --- | --- |
| `<red>badJson</red>` | `400` | None | The JSON body could not be parsed. |
| `<red>badBody</red>` | `400` | `{ reason: string }` | Body, query, params, or form data validation failed. |
| `<red>badToken</red>` | `401` | None | The auth token is missing or invalid. |
| `<red>badPerms</red>` | `403` | None | The authenticated user is missing required permission bits. |
| `<red>badEndpoint</red>` | `404` | None | The route or provider-backed endpoint is unavailable. |
| `<red>badNotStarted</red>` | `401` | None | The route is gated until CTF start time. |
| `<red>badEnded</red>` | `401` | None | The action is gated after CTF end time. |
| `<red>badCaptcha</red>` | `403` | None | V2 captcha validation failed. |
| `<red>badRecaptchaCode</red>` | `401` | None | V1 captcha validation failed. |
| `<red>badRateLimit</red>` | `429` | `{ timeLeft: number }` | The route-specific rate-limit bucket is exhausted. |
| `<red>errorInternal</red>` | `500` | None | An unhandled server error occurred. |

## Shared object shapes

### `ChallengeFileV2`

| Field  | Type             |
| ------ | ---------------- |
| `name` | `string`         |
| `url`  | `string`         |
| `size` | `number \| null` |

### `Endpoint`

| Field   | Type                                      |
| ------- | ----------------------------------------- |
| `kind`  | `"tcp" \| "tcp-ssl" \| "http" \| "https"` |
| `host`  | `string`                                  |
| `port`  | `number`                                  |
| `title` | `string \| undefined`                     |

### `InstancerConfig`

| Field                    | Type                      |

| `challengeIntegrationId` | `string`                  |

| `extendable`             | `boolean \| undefined`    |
