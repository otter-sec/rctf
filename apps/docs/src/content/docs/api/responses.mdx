title: Responses
description: Reference for rCTF response envelopes, common errors, response payloads, and shared API object shapes.
---

Response definitions live in `@rctf/types` under `packages/types/src/responses`. Each definition fixes the response `kind`, HTTP status, message, and optional `data` schema.

## Envelope

Responses with data use:

```json
{
  "kind": "goodUserSelfData",
  "message": "The user's own data was successfully retrieved.",
  "data": {}
}
```

Responses without data omit `data`:

```json
{
  "kind": "goodToken",
  "message": "The authorization token is valid."
}
```

Global 404 errors return `badEndpoint`. Unhandled route errors return `errorInternal`.

## Common errors

| Kind | Status | Data | Meaning |
| --- | --- | --- | --- |
| `badJson` | `400` | None | The JSON body could not be parsed. |
| `badBody` | `400` | `{ reason: string }` | Body, query, params, or form data validation failed. |
| `badToken` | `401` | None | The auth token is missing or invalid. |
| `badPerms` | `403` | None | The authenticated user is missing required permission bits. |
| `badEndpoint` | `404` | None | The route or provider-backed endpoint is unavailable. |
| `badNotStarted` | `401` | None | The route is gated until CTF start time. |
| `badEnded` | `401` | None | The action is gated after CTF end time. |
| `badCaptcha` | `403` | None | V2 captcha validation failed. |
| `badRecaptchaCode` | `401` | None | V1 captcha validation failed. |
| `badRateLimit` | `429` | `{ timeLeft: number }` | The route-specific rate-limit bucket is exhausted. |
| `errorInternal` | `500` | None | An unhandled server error occurred. |
