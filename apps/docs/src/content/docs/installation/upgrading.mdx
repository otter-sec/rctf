This guide covers the changes between rCTF v1 and v2 and what you need to do to upgrade.

## Database migration

Database schema changes are handled automatically by Drizzle ORM. Run migrations before starting the new version:

```ansi
$ [31mbun[0m run db:migrate
```

Or set `[31mdatabase.migrate[0m{:ansi}` to `[32mbefore[0m{:ansi}` in your configuration to migrate on startup.

### New tables

| Table            | Purpose                                                                    |
| ---------------- | -------------------------------------------------------------------------- |
| `admin_bot_jobs` | Tracks admin bot job queue (status, inputs, logs)                          |
| `settings`       | Stores runtime-editable settings (CTF name, client timing, sponsors, etc.) |

### New columns on `users`

| Column                   | Type      | Description                          |
| ------------------------ | --------- | ------------------------------------ |
| `avatar_url`             | text      | URL to user's avatar image           |
| `country_code`           | text      | ISO 3166-1 alpha-2 country code      |
| `status_text`            | text      | User status message (max 60 chars)   |
| `score`                  | integer   | Cached current score                 |
| `global_rank`            | integer   | Cached global leaderboard rank       |
| `division_rank`          | integer   | Cached division-specific rank        |
| `last_solve_at`          | timestamp | Time of last solve (for tiebreaking) |
| `last_tiebreak_solve_at` | timestamp | Time of last tiebreak-eligible solve |

### New columns on `challenges`

| Column        | Type    | Description                    |
| ------------- | ------- | ------------------------------ |
| `score`       | integer | Cached current challenge score |
| `solve_count` | integer | Cached number of solves        |

## API changes

All v1 routes continue to work. The v2 API is available at `[2m/api/[22mv2[2m/[22m...{:ansi}` and includes several improvements and new endpoints.

### Captcha field rename

V2 routes use `[31mcaptchaCode[0m{:ansi}` instead of `[31mrecaptchaCode[0m{:ansi}` in request bodies. This applies to all endpoints that support captcha validation.

### File uploads

The v2 admin upload endpoint (`[32mPOST[0m [2m/api/[22mv2[2m/[22madmin[2m/[22mupload{:ansi}`) accepts binary files via `multipart/form-data` instead of base64-encoded data URIs used in v1.

### Response differences

- Challenge files in v2 include a `size` field (v1 did not track file sizes)
- Challenge solves in v2 include user profile data (avatar, country, status) and blood status
- Leaderboard in v2 supports a `search` query parameter for team name search (rate limited)

## Configuration changes

### Captcha provider

The v1 `[31mrecaptcha[0m{:ansi}` top-level config is automatically converted to the new `[31mcaptcha.provider[0m{:ansi}` format at startup. However, migrating to the new format is recommended:

<Tabs items={['v1 (deprecated)', 'v2 (recommended)']}>
<Tab>

```yaml title="v1"
recaptcha:
  siteKey: your-site-key
  secretKey: your-secret-key
  protectedActions:
    - register
```

</Tab>
<Tab>

```yaml title="v2"
captcha:
  provider:
    name: captcha/recaptcha
    options:
      siteKey: your-site-key
      secretKey: your-secret-key
  protectedEndpoints:
    - register
```

</Tab>
</Tabs>

The new format also supports `[32mcaptcha/hcaptcha[0m{:ansi}` and `[32mcaptcha/turnstile[0m{:ansi}` providers.

### Analytics provider

The v1 `[31mglobalSiteTag[0m{:ansi}` config is automatically converted to the `[31manalytics.provider[0m{:ansi}` format. Migrate to:

```yaml title="v2 (recommended)"
analytics:
  provider:
    name: analytics/google
    options:
      siteTag: G-XXXXXXXX
```

Additional analytics providers are available: `[32manalytics/cloudflare[0m{:ansi}` and `[32manalytics/plausible[0m{:ansi}`.

### New configuration sections

The following configuration sections are new in v2:

| Section                    | Purpose                                      |
| -------------------------- | -------------------------------------------- |
| `[31madminBot[0m{:ansi}`          | Admin bot provider and settings              |
| `[31mavatarsModeration[0m{:ansi}` | Avatar content moderation (OpenAI)           |
| `[31mbloodBot[0m{:ansi}`          | First blood notifications (Discord/Telegram) |
| `[31manalytics[0m{:ansi}`         | Analytics provider                           |
| `[31mproxy[0m{:ansi}`             | Reverse proxy and Cloudflare settings        |
| `[31mmaxAvatarSize[0m{:ansi}`     | Maximum avatar upload size (default 1 MB)    |

See the [Configuration](/configuration/) reference for full details on all options.
