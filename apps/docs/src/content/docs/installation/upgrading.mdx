This guide covers the changes between rCTF v1 and v2 and what you need to do to upgrade.

## Database migration

Database schema changes are handled automatically by Drizzle ORM. Run migrations before starting the new version:

```bash title="Terminal"
bun run db:migrate
```

Or set `database.migrate{:yaml}` to `before{:yaml}` in your configuration to migrate on startup.

### New tables

| Table            | Purpose                                                                    |
| ---------------- | -------------------------------------------------------------------------- |
| `admin_bot_jobs` | Tracks admin bot job queue (status, inputs, logs)                          |
| `settings`       | Stores runtime-editable settings (CTF name, client timing, sponsors, etc.) |

### New columns on `users`

| Column                   | Type      | Description                          |
| ------------------------ | --------- | ------------------------------------ |
| `avatar_url`             | text      | URL to user's avatar image           |
| `country_code`           | text      | ISO 3166-1 alpha-2 country code      |
| `status_text`            | text      | User status message (max 60 chars)   |
| `score`                  | integer   | Cached current score                 |
| `global_rank`            | integer   | Cached global leaderboard rank       |
| `division_rank`          | integer   | Cached division-specific rank        |
| `last_solve_at`          | timestamp | Time of last solve (for tiebreaking) |
| `last_tiebreak_solve_at` | timestamp | Time of last tiebreak-eligible solve |

### New columns on `challenges`

| Column        | Type    | Description                    |
| ------------- | ------- | ------------------------------ |
| `score`       | integer | Cached current challenge score |
| `solve_count` | integer | Cached number of solves        |

## API changes

All v1 routes continue to work. The v2 API is available at `/api/v2/...` and includes several improvements and new endpoints.

### Captcha field rename

V2 routes use `captchaCode{:yaml}` instead of `recaptchaCode{:yaml}` in request bodies. This applies to all endpoints that support captcha validation.

### File uploads

The v2 admin upload endpoint (`POST /api/v2/admin/upload`) accepts binary files via `multipart/form-data` instead of base64-encoded data URIs used in v1.

### New v2 endpoints

| Endpoint                                                    | Description                                           |
| ----------------------------------------------------------- | ----------------------------------------------------- |
| `GET /v2/auth/verify-info`                                  | Returns info about a verification token               |
| `GET /v2/leaderboard/challs`                                | Challenge metadata with first 3 solvers per challenge |
| `GET /v2/leaderboard/with-graph`                            | Combined leaderboard and graph in one request         |
| `PATCH /v2/users/me/avatar`                                 | Upload and moderate avatar images                     |
| `GET /v2/admin/users`                                       | List all users with search and pagination             |
| `GET/PUT /v2/admin/settings`                                | Runtime settings management                           |
| `POST /v2/admin/users/:id/token`                            | Generate login token for a team                       |
| `DELETE /v2/admin/challs/:id/solves/:userId`                | Delete a specific solve                               |
| `GET /v2/admin/instancer/schema`                            | Instancer config JSON schema                          |
| `GET/POST /v2/admin/admin-bot/*`                            | Admin bot job management                              |
| `GET/PUT/PATCH/DELETE /v2/integrations/challs/:id/instance` | Challenge instance lifecycle                          |
| `GET/POST /v2/integrations/challs/:id/admin-bot/*`          | Admin bot submission and status                       |

### Response differences

- Challenge files in v2 include a `size` field (v1 did not track file sizes)
- Challenge solves in v2 include user profile data (avatar, country, status) and blood status
- Leaderboard in v2 supports a `search` query parameter for team name search (rate limited)

## Configuration changes

### Captcha provider

The v1 `recaptcha{:yaml}` top-level config is automatically converted to the new `captcha.provider{:yaml}` format at startup. However, migrating to the new format is recommended:

<Tabs items={['v1 (deprecated)', 'v2 (recommended)']}>
<Tab>

```yaml title="v1"
recaptcha:
  siteKey: your-site-key
  secretKey: your-secret-key
  protectedActions:
    - register
```

</Tab>
<Tab>

```yaml title="v2"
captcha:
  provider:
    name: captcha/recaptcha
    options:
      siteKey: your-site-key
      secretKey: your-secret-key
  protectedEndpoints:
    - register
```

</Tab>
</Tabs>

The new format also supports `captcha/hcaptcha{:yaml}` and `captcha/turnstile{:yaml}` providers.

### Analytics provider

The v1 `globalSiteTag{:yaml}` config is automatically converted to the `analytics.provider{:yaml}` format. Migrate to:

```yaml title="v2 (recommended)"
analytics:
  provider:
    name: analytics/google
    options:
      siteTag: G-XXXXXXXX
```

Additional analytics providers are available: `analytics/cloudflare{:yaml}` and `analytics/plausible{:yaml}`.

### New configuration sections

The following configuration sections are new in v2:

| Section                    | Purpose                                      |
| -------------------------- | -------------------------------------------- |
| `adminBot{:yaml}`          | Admin bot provider and settings              |
| `avatarsModeration{:yaml}` | Avatar content moderation (OpenAI)           |
| `bloodBot{:yaml}`          | First blood notifications (Discord/Telegram) |
| `analytics{:yaml}`         | Analytics provider                           |
| `proxy{:yaml}`             | Reverse proxy and Cloudflare settings        |
| `maxAvatarSize{:yaml}`     | Maximum avatar upload size (default 1 MB)    |

See the [Configuration](/configuration) reference for full details on all options.
