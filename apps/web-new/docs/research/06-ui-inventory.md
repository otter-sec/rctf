# rCTF Web Component Layer Inventory (apps/web/src)

Counts = files importing/using outside `lib/components/ui/**` (barrel is `lib/components/index.ts`; all app code imports via `$lib/components`, tag usage counted as `<Family.X` / `<Name`). Route areas: **auth** (login/register/recover/verify/external-auth), **chal** (challenges), **scores**, **prof** (profile, profile/[id]), **admin** (challenges/settings/submissions/teams/profile-[id]), **chrome** (lib/components navigation etc.), **home/root**.

Third-party in ui layer: bits-ui (80 imports, 17 families), vaul-svelte (drawer), paneforge (resizable), svelte-sonner (toasts), @tanstack/svelte-virtual (virtual-list), tailwind-variants (variant helpers), layerchart (charts, imported in routes). All styling via `cn()` + Tailwind classes — all of that goes away.

## Dead / near-dead weight

- **Zero usage, delete outright:** button-group, input-group, pagination, sidebar (26 files), skeleton, table, separator (external use = 0; only intra-ui). None referenced anywhere in routes/lib.
- **Single-consumer, fold or replace with native:** badge (1), label (1 file), progress (1), checkbox (1 file), sheet (1 = mobile nav), popover (1 = flag-picker), command (1 = flag-picker), `command-dialog.svelte` (0 — no command palette exists despite the family shipping one).
- `badgeVariants`/`buttonVariants` helpers: only external consumer is brainrot.svelte.

## Top 10 hardest problems

1. **Scores leaderboard virtualized grid** (routes/scores/\*, bespoke — does NOT use ui/virtual-list): own windowing via `scores-leaderboard-scroll-state` + fixed row heights, scroll fades, sticky headers, per-cell solve tooltips, sparklines, delta indicators, self-row pinning. Rec: port as-is with scoped CSS; it's already independent of shadcn/bits; only Tooltip/Checkbox/Dialog leaf swaps needed.
2. **Charts** (9 layerchart components: scores graph, chal scores graph, profile timeline/category/difficulty/cadence/graph, nav + row sparklines): rec: keep layerchart; replace chart-container's Tailwind arbitrary-selector overrides of `.lc-*` internals with `:global` scoped CSS; audit theme CSS-variable plumbing (fadeColor etc. reference `var(--background-l0)`-style tokens).
3. **schema-form**: rec: port wholesale (logic is dependency-free); re-skin the 16 files with native CSS; swap leaf inputs (Input/Textarea/Select/Checkbox/TagInput) to new primitives; keep validate.ts + context error registry untouched.
4. **Tooltip at scale** (32 files, provider delay-grouping incl. scores' nested skip-delay=0 region): rec: Zag tooltip + thin Root/Trigger/Content wrapper preserving `child` snippet prop-merging and `disableCloseOnTriggerClick`; verify Zag's global-store grouping matches bits-ui Provider semantics before mass migration; @floating-ui/dom for positioning.
5. **Dropdown menu w/ submenus + checkbox items** (admin filter system, 9 files): rec: Zag menu with nested machines; build once, validated against admin/submissions/filters (its most demanding consumer).
6. **Dialog/Sheet/Drawer unification** (3 libs today: bits-ui dialog, bits-ui dialog-based sheet, vaul-svelte drawer): rec: one primitive (native `<dialog>` or Zag dialog) with three CSS presentations (center/side/bottom); explicitly decide fate of vaul's drag-to-dismiss (small pointer handler or drop).
7. **Virtualized lists** (ui/virtual-list + lib/utils/virtualizer.svelte.ts): rec: keep @tanstack/svelte-virtual and the custom rect/offset observers (they solve real late-mount/polling problems); only restyle the 59-line renderer.
8. **flag-picker combobox** (sole Command+Popover consumer, ~250 regions, images, clear option, focus return): rec: Zag combobox; this single migration deletes two families.
9. **ScrollArea with edge fades** (22 consumers, focusable viewports, per-edge fade offsets): rec: native `overflow:auto` + CSS scrollbar styling; keep the existing RAF/ResizeObserver fade code; expose same `viewportRef`/`viewportTabIndex`/`fadeOffsets` API so consumers port mechanically.
10. **Toast system** (28 call-site files using `toast.*` from svelte-sonner): rec: Zag toast group behind a sonner-compatible `toast.success/error/info/warning/loading` shim (theme-aware via `data-theme`); or consciously keep svelte-sonner since it's independent of tailwind/bits — cheapest correct option if dependency count allows.

Also nontrivial but bounded: Resizable → **Zag splitter** (2 split-pane screens, keyboard a11y); Accordion → **Zag accordion** with controlled multi-value for collapse-all; Select → **Zag select** with snippet-based item rendering (67 tags); markdown island hydration (keep `mount()` pattern).
