## 4. Repo conventions & typecheck setup

- **Runtime: Bun** (v1.0+, Docker pins `oven/bun:1.3.14-alpine`), workspaces monorepo (`apps/*`, `packages/*`, `tests/*`). Not pnpm/node — scripts use `bunx --bun`.
- **CONTRIBUTING** (`.github/CONTRIBUTING.md`): pre-PR checks = `bun run test`, `bun run typecheck`, `bun run prettier:check`. Conventional-commit PR titles scoped by workspace (`fix(web): ...`); squash merges. Behavior changes must update `apps/docs` in the same PR. No AGENTS.md/CLAUDE.md in repo root or apps/web.
- **Typecheck:** `apps/web` typecheck = `svelte-kit sync && svelte-check --tsconfig ./tsconfig.json`. Chain: `apps/web/tsconfig.json` → extends generated `.svelte-kit/tsconfig.json`, and `svelte.config.ts` rewrites the generated config's `extends` to the **root** `/tsconfig.json`.
- **Root tsconfig strictness:** `strict`, `noUncheckedIndexedAccess`, `noImplicitOverride`, `noFallthroughCasesInSwitch`, `verbatimModuleSyntax`, `isolatedModules`, `moduleResolution: bundler`, `module: Preserve`, `allowImportingTsExtensions`, `checkJs`. Notably **disabled**: `noUnusedLocals`, `noUnusedParameters`, `noPropertyAccessFromIndexSignature`. No `exactOptionalPropertyTypes`.
- **Path aliases:** only SvelteKit defaults — `$lib`, `$lib/*`, `$app/*` (from generated tsconfig). No custom aliases; workspace imports via `@rctf/types`, `@rctf/util` (`workspace:*`).
- **CI** (`.github/workflows/_test.yml`): separate jobs for `prettier:check`, `typecheck`, server tests, admin-bot tests, admin-bot e2e. No web-specific unit tests exist. `ci.yml` uses paths-filter with `apps/web/**` gating Docker builds; actions pinned to SHAs.

## 5. Deployment / serving model (constrains web-new backend-URL resolution)

- Single container (`deploy/rctf/Dockerfile`): **supervisord** runs (a) Bun API on `127.0.0.1:3000` and (b) **nginx** on port 80. Web is built with `vite build` → `apps/web/build` → copied to `/app/static` and served as **static files by nginx** with `try_files ... /index.html` SPA fallback.
- **adapter-static** with `fallback: 'index.html'`, `precompress: true` (nginx serves `gzip_static`/`brotli_static`), `strict: true`.
- **Backend URL is always same-origin**: nginx proxies `^/(api|uploads)` to the API upstream; special-case `location ~ ^/api/v[12]/admin/upload$` with unlimited body size and no request buffering (large file uploads). Dev: vite proxies `/api` and `/uploads` to `http://127.0.0.1:3000` (`DEV_API_URL` in `vite.config.ts`). The API client builds URLs as `new URL('/api/'+path, browser ? window.location.origin : 'http://localhost')` — **no env injection at deploy time; web-new must keep same-origin relative `/api` paths** (all runtime config comes from the API's client-config endpoint; server-side `rctf.d/*.yaml` holds `origin`, divisions, captcha, etc.).
- **Headers/caching:** nginx adds `Referrer-Policy: no-referrer`, `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`; `/_app/immutable/` gets `Cache-Control: public, max-age=86400, immutable`. **CSP is generated by SvelteKit as a `<meta>` tag** with auto script hashes (prod only; see `svelte.config.ts` for the full directive list covering hcaptcha/recaptcha/turnstile, GCS/S3, GA/Cloudflare analytics, YouTube frames). web-new must preserve this CSP mechanism — `frame-ancestors` intentionally lives in nginx, not the meta CSP.
- `compose.yml`: rctf image + redis 8.2.2 + postgres 18.0; config mounted at `/app/rctf.d`, uploads at `/app/uploads`; app exposed at `127.0.0.1:8080:80`. `compose.dev.yml` only runs redis+postgres (dev servers run on host via `bun dev`).
